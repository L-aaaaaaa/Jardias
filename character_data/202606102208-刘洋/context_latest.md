## 三、进一步支持商业秘密的五个关键理由

- **《专利法》第 2 条第 2 款**：未构成技术方案（缺乏技术性）

尽管我们已经用工程化语言包装（"步骤 S101-S105""状态机""数据结构"），但审查员仍可能认定核心是"管理 Agent 记忆的规则"，不构成技术方案。**驳回风险客观存在**，一旦驳回则既没有专利、又因公开而丧失商业秘密。

| 申请法院调查取证 | ❌ 实践中极少获批 |

### 具体建议

| 保护方式 | 内容 | 理由 |
|----------|------|------|
| **商业秘密（主）** | 复合评分权重、蒸馏编排策略、InfoStatus 转换规则、容量阈值参数 | 反向工程难度极高，是最有商业价值的差异化核心 |
| **商业秘密 + 选择性开源** | 九维 MECE 数据模型 | 开源以建立社区标准，但训练/优化的具体策略保密 |
| **防御性专利（1-2 件）** | 底层方法流程（S101-S105 的顶层步骤） | ① 防止竞争对手"反向申请"反制我们；② 保留"专利技术"市场宣传标签；③ 融资/估值时有专利资产 |

## 五、如果选择商业秘密，需要做什么？

商业秘密不是"不申请专利就行"，而是需要**主动采取保密措施**：

| 措施 | 具体要求 |
|------|---------|
| **制度层面** | 制定《商业秘密管理制度》，明确保密范围和密级 |
| **技术层面** | 核心参数与代码分离（参数走配置中心加密存储）、访问权限分级 |
| **人员层面** | 核心研发人员签署保密协议 + 竞业限制协议 |
| **物理层面** | 代码仓库权限管控、内部文档标注密级 |
| **留存证据** | 保密培训记录、保密协议原件、访问日志（为将来可能的诉讼留证） |

> ⚠️ 如果未采取保密措施，即使不申请专利，法律上也不认定为商业秘密。

> **HEEL 的工程实例化内容，80% 适合商业秘密，15% 适合防御性公开/开源，5% 可做防御性专利申请。** 尤其是在 AI Agent 这一快速迭代、内部逻辑不易反向的领域，商业秘密是比专利更务实、更有效、风险更低的选择。但需要配套建立保密制度——"不申请专利"不等于"受商业秘密保护"。
