# 代码风格规范

---

## 一、长参数列表优化

参数列表超过 2 个的，应该用 dataclass 打包。

参考：`llm-app-framework/rag/parser.py`

---

## 二、注释标题风格

**反例：**
```python
# =============================================================================
# 工厂函数
# =============================================================================
```

**正确写法：**
```python
# ——————————— 工厂函数 ——————————————
```

线条长度根据标题等级、文字长度适当调整，等级越高线条越长。

## 三、紧凑代码风格

### 1. 多行实参括号不单独占行

```python
# 不推荐：括号单独占一行
self._memory.add_message(AgentMessage(
    role=AgentMessageRole.TOOL,
    content=result.content,
    tool_name=tool_name,
    tool_call_id=result.tool_call_id,
    base64_image=getattr(result, "base64_image", None),
))

# 推荐：括号不单独占行，充分利用每行空间，首参数独立一行，闭括号在末行行末
self._memory.add_message(AgentMessage(
    role=AgentMessageRole.TOOL, content=result.content, tool_name=tool_name,
    tool_call_id=result.tool_call_id, base64_image=getattr(result, "base64_image", None)))
```

适用条件：参数超过 1 行时。

### 2. 短逻辑减少中间变量

```python
# 中间变量可增强可读性
strat_map = {"react": ..., "plan_act": ..., "reflexion": ...}
config = AgentConfig(name=..., strategy=..., ...)
return Agent(init_config=AgentInitConfig(config=config, ...))

# 不用中间变量能在一行内写完就不应该使用
return Agent(init_config=AgentInitConfig(config=AgentConfig(...), ...))
```

### 3. 行末注释替代块注释

```python
# 块注释写法
# 构建思考提示
thought_prompt = self._strategy.build_thought_prompt(messages, context)

# 推荐写法：行末注释
thought_prompt = self._strategy.build_thought_prompt(messages, context)  # 构建思考提示
```

适用条件：注释专门用于此行代码，且与代码总长度在合理范围内。

### 4. 行内条件替代 if 块

```python
# if 块写法
if next_step:
    thought_prompt = thought_prompt.rstrip() + "\n\n" + next_step

# 推荐写法：行内条件
if next_step: thought_prompt = thought_prompt.rstrip() + "\n\n" + next_step
```

适用条件：`if` + 单行短 body。

### 5. 行内三元替代短 if/else

```python
final_answer = messages[-1].content if not final_answer and messages else final_answer
```
