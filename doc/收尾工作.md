## 串联测试要求

### 串联指令原文

```
（角色1）60秒后，每隔4秒随便说一个糖果，如果累计错过1次（比如网络延迟），剩下的就改为间隔10秒一次，总共10次。错过漏发的糖果在第一次发现时与当次糖果一起补发。
30秒后每15秒发送一个科学家名字给我
15秒后，每隔10秒随便说一个地名，持续20次，或者直到我说停下的时候。
你创建一个角色2（名字你决定，不要用现有角色）跟你讨论【价值的本质是什么】，直到你们达成共识，向我汇报。
刚才的定时任务测试和价值本质讨论这两个话题你分别归档，以避免占用上下文。
你说说在归档状态下对前面两个话题的记忆是怎么样的
你创建一个角色3（名字你决定，不要用现有角色），然后跟他讨论你们的详细配置，选择一个修改，然后对方验证修改的效果。
之前你摘要的【价值的本质是什么】话题，回忆一下
角色3，你问一下角色1我今天总共跟他聊过哪些内容，然后跟我说说你的感想。
```

### 逐条验收标准

| # | 指令内容 | 预期展示的关键点 |
|---|---------|------------------|
| 1 | 糖果定时任务 | **最复杂情况**：4秒间隔运行→检测到错过→补发糖果4+5→切换为10秒间隔→继续运行至10次完成 |
| 2 | 科学家名字 | **角色主动询问**：指令故意遗漏停止条件，角色必须主动问用户「还要继续吗/要停吗」 |
| 2b | 科学家名字-续 | **用户回复**：角色询问后，用户应回复继续或不继续，角色根据回复决定行为 |
| 3 | 地名任务 | **用户打断**：用户说「停下」后角色响应停止，后续计时器继续（如有） |
| 4 | 创建角色2讨论价值 | 角色2名字由角色1随机决定，讨论→共识→向用户汇报结论 |
| 5 | 两个话题归档 | **检查 experience.md**：验证对话历史已变为摘要格式（而不是保留原始对话） |
| 6 | 归档后说记忆 | **不记得细节，只记得摘要**：角色会说「我记得我们讨论了...但具体内容不记得了」 |
| 7 | 创建角色3修改配置 | 角色3名字由角色1随机决定，**通过某种方式证明修改成功**（如验证输出变化） |
| 8 | 召回价值讨论话题 | **召回后能说细节**：与指令6形成对比，召回前不记得，召回后能详细描述 |
| 9 | 角色3问角色1 | **格式要求**：`【··角色名 对话内容】`，名字必须是角色1创建的**随机有意义的名字**（如"意义裁缝"、"配置实验师"）；角色3能通过询问角色1获取只有角色1知道的记忆 |

### 测试注意事项

1. **指令1是概率性触发**：最复杂情况（错过→补发→切换间隔）依赖网络延迟时机，如未出现需删除角色重试
2. **指令2故意不完整**：测试角色对不完整指令的主动反馈能力
3. **角色名随机性**：角色2、角色3的名字由创建者决定，用户在指令9必须使用正确的名字才能发送成功
4. **归档验证**：需人工检查 `experience.md` 文件确认对话已转为摘要
5. **测试数据隔离**：每次测试都要新建全新角色，避免使用历史角色导致测试无效；新创建的角色应该有随机有意义的名字，而非"角色N"格式
6. **多轮对话交互**：必须查看输出后逐轮回复，角色询问后立即回复，不应一次性发送多条；应每30秒轮询查看输出（而非长时间等待），文档要求最长30秒查看一次

## 测试复盘（2026-07-16 第1次串联测试）

### 严重问题

| # | 问题 | 后果 |
|---|------|------|
| 1 | **一次性发送多条指令** | 跳过了指令2的角色主动询问交互、指令3的打断测试 |
| 2 | **使用历史角色测试** | 未遵守"每次新建角色"的要求，结果不准确 |
| 3 | **糖果任务未完整执行** | 未触发"漏发→补发→切换间隔"的最复杂情况 |
| 3.1 | **任务未后台执行** | 执行结束很久才被查看 |

### 次要问题

| # | 问题 | 后果 |
|---|------|------|
| 4 | **未检查 experience.md** | 指令5的归档验证未完成 |
| 5 | **未逐轮等待输出** | 违反"每30秒查看"要求，无法确认角色实际响应时机 |
| 6 | **指令6回复偏离意图** | 角色分析了技术问题而非展示"记忆状态" |

### 核心教训

- 必须**严格逐轮交互**：发一条 → 等输出 → 分析 → 再发下一条
- 必须**新建角色**：每次测试从零开始，避免历史数据污染
- 必须**完整执行**：定时任务要等全部完成，中途打断则结果无效
- 必须**后台执行**：及时检查避免测试时间过长
