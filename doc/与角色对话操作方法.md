# 外部智能体与佳递叶思角色对话的方法

本文档介绍如何通过命令行自动化与 Jardias 智能体框架中的角色进行对话。

## 方法：cmd 文件重定向（**不要尝试PowerShell 方式**）

*PowerShell 的 < 重定向与某些参数组合会出问题，应该用 cmd /c 来执行。*

### 步骤 1：编辑输入文件

`tests/test_input.txt`

```
20
你好
请介绍一下你自己
```

**格式说明：**
- 第 1 行：选择角色编号（如 `20` 选择"测试员3"）
- 第 2 行：第一次对话内容
- 第 3 行：第二次对话内容
- 以此类推...

### 步骤 2：运行（输出保存到 logs 目录）

```bash
cd e:/Code/AIProjects/Jardias
cmd /c "D:\B\Python3.10\python.exe app.py < tests\test_input.txt" > logs\output_时间戳.txt 2>&1
```

**说明：**
- `2>&1` 把错误输出也一起保存
- 输出文件保存在 `logs/` 目录，文件名建议带时间戳便于区分
- 文件中会保留 ANSI 颜色码（见下方"颜色代码"说明）

**后台运行方式**（不保存到文件，用于实时观察）：
```bash
# 直接运行，输出在当前终端文件夹的 Shell ID.txt 中（含颜色码）
cmd /c "D:\B\Python3.10\python.exe app.py < tests\test_input.txt"
```

## 查看对话输出

查看 `logs/output_*.txt` 文件即可，无需后处理。

### 清理 ANSI 颜色码（可选）

Python 输出包含 `\033[33m` 等转义码，肉眼阅读可忽略。如需纯文本，可用 `tests/strip_ansi.py`：

```bash
# 单个文件 - 直接打印清理后的内容到 stdout
python tests/strip_ansi.py logs/output_202607141818.txt

# 批量 + 就地修改（去除颜色码后回写原文件）
python tests/strip_ansi.py logs/*.txt --inplace
```

脚本逻辑：用正则 `\x1b\[[0-9;]*[A-Za-z]` 匹配所有 ANSI CSI 序列并删除。

## 查看角色列表

运行后在输出开头会显示所有角色及其编号。

## 停止运行

如果对话卡住或需要中断：

```powershell
Stop-Process -Name python -Force
```

或指定 PID（Shell 输出中可见）：
```powershell
Stop-Process -Id <PID> -Force
```

## 等待与轮询策略

### 问题
AI 代理调用通常只需要几秒到十几秒，但设置过长超时（如 31秒）会导致不必要等待。

### 推荐做法
- **后台运行 + 轮询检查更高效**：用 5-10 秒超时，配合循环检查输出文件
- **检测完成标志**：等待日志中出现 `本轮输入` 或 `轮到你了` 等标志
- **不要设置 30+ 秒超时**：最长30秒应该查看一次

### 示例（Python）

```python
import time
from pathlib import Path

def wait_for_completion(output_file: Path, max_attempts=12, interval=5):
    """等待命令完成，最多等 max_attempts * interval 秒"""
    for _ in range(max_attempts):
        time.sleep(interval)
        content = output_file.read_text(encoding='utf-8', errors='replace')
        if '本轮输入' in content or '轮到你了' in content:
            return True
    return False
```

## 注意事项

1. **编码**：输入文件使用 UTF-8 编码
2. **多轮对话**：每行是一次回车，按顺序执行
3. **颜色代码**：输出文件会保留 ANSI 颜色码（`\033[33m` 等），这是 Python 直接写入的，重定向无法去除；不影响肉眼阅读，grep/分析时可忽略
4. **角色名冲突**：创建前确保名字在角色列表中唯一
5. **不必死板创建 txt 文件**：对于简单的单轮对话，可以直接用 `echo` 或 PowerShell 管道传入内容，无需预先创建文件。例如：`echo 你好 | cmd /c "python app.py --character 角色名"`

## 创建新角色

在角色选择时输入 `N` 进入创建流程。

### 创建流程的真实提示顺序（实测）

输入文件 `tests/create_char.txt` 需按以下顺序提供答案，每行一个：

```
N
<角色名>
<特质描述（可空，直接回车）>
<头衔（可空，直接回车）>
<系统提示：1=默认助手 / 2=代码助手 / 3=自定义>
<引擎：1-N（具体编号由 providers.json 决定）>
<对话内容 1>
<对话内容 2>
...
quit
```

**实测输入示例**（`tests/create_char.txt`）：
```
N
测试员5
严谨认真的测试工程师
测试专家
1
3
你好
请介绍一下你自己
quit
```

**示例解释：**
- 第 1 行：`N` 触发创建
- 第 2 行：角色名 `测试员5`
- 第 3 行：特质描述
- 第 4 行：头衔
- 第 5 行：选择系统提示 `1` = 默认 AI 助手
- 第 6 行：选择引擎 `1` = minimax/2.7快（包月账户，优先使用）
- 第 7-8 行：与新角色的对话内容
- 第 9 行：`quit` 退出

### 运行命令

```bash
cmd /c "D:\B\Python3.10\python.exe app.py < tests\create_char.txt" > logs\create_时间戳.txt 2>&1
```

### 创建后的影响

1. 角色文件夹立即在 `character_data/{YYYYMMDDHHMM}-角色名/` 下创建
2. 包含 `config.json`、`history.json`、`experience.md`、`summaries/L1/`
3. 下次运行可直接用编号选择新角色

### 注意事项

1. **创建后必然进入对话循环** - 文件末尾必须跟足够对话行 + `quit`，否则会卡在 `input()` 等待
2. **角色名重复会报错** - `ValueError: 角色 XXX 已存在`，需先删除 `character_data/{时间戳}-XXX/`
3. **引擎编号动态变化** - 由 `yinao/providers.json` 决定，1-N 对应各 provider/IPU 组合
4. **引擎计费方式（优先使用包月引擎）：**
   - `minimax` → **包月**，优先使用
   - `deepseek` → 按量计费
   - `dashscope` → 欠费，慎用
5. **输出含 ANSI 颜色码** - 重定向文件会保留 `\033[33m` 等转义码，肉眼阅读可忽略

## 多轮对话的方法

### 目的

实现与角色讨论，根据角色的回复决定下次消息。比如非硬编码的成语接龙、话题辩论反驳、模拟真实用户使用项目的过程。

### 单轮批处理

核心原理：每次运行只发送一轮输入，通过 `--character` 参数跳过角色选择菜单。`history.json` 会自动保存对话历史。

**第一步：首次运行 - 创建角色并开始对话**

```powershell
# 创建输入文件
$N | Out-File -FilePath tests/create_and_chat.txt -Encoding utf8
"成语大师" | Add-Content tests/create_and_chat.txt
"热爱成语，擅长文字游戏" | Add-Content tests/create_and_chat.txt
"" | Add-Content tests/create_and_chat.txt  # 头衔留空
"1" | Add-Content tests/create_and_chat.txt  # 默认系统提示
"1" | Add-Content tests/create_and_chat.txt  # 默认引擎
"你好，我们玩成语接龙，我先说：天下无双" | Add-Content tests/create_and_chat.txt
"quit" | Add-Content tests/create_and_chat.txt

# 运行
python app.py < tests/create_and_chat.txt > logs/round1.txt 2>&1
```

**第二步：查看输出，决定下一轮**

从 `logs/round1.txt` 中提取角色的回复，确定角色说了什么成语。

**第三步：继续运行 - 只需提供新输入**

```powershell
# 方法1：直接用 echo（推荐用于简单场景）
echo 下落不明 | cmd /c "D:\B\Python3.10\python.exe app.py --character 成语大师" > logs\round2.txt 2>&1

# 方法2：先创建输入文件（适合复杂或多轮对话）
"下落不明" | Out-File tests/next_input.txt -Encoding utf8
python app.py --character "成语大师" < tests/next_input.txt > logs/round2.txt 2>&1
```

**第四步：重复**

不断重复"查看输出 → 决定回复 → 运行"的循环。

**完整示例（成语接龙3轮）：**

```
# 轮次1
输入: 你好，我们玩成语接龙，我先说：天下无双
输出: 双管齐下（需要接"下"字开头）

# 轮次2
输入: 下落不明
输出: 明明白白（需要接"白"字开头）

# 轮次3
输入: 白手起家
输出: 家喻户晓
```


### 关键原理

1. **history.json 自动保存**：每次对话后，`history.json` 会保存所有历史记录
2. **重新运行自动加载**：下次运行时会自动加载历史，继续上下文
3. **--character 参数**：跳过选择菜单，直接使用指定角色
4. **单轮输入**：后续轮次只需提供新的用户输入，不需要重复历史
