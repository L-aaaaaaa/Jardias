### 0.1 The Predicament of the Point: Logical-Type Confusion

If any information-processing system permits two categories belonging to distinct logical types to share the same "point"---the same representational space, the same field, the same scalar---at the data layer, it commits **logical-type confusion**.

Mathematically, a point $(x_1, y_1)$ is a coupling. What the input is, what the output is, and what the relationship between them is are all compressed into the same coordinate. One cannot read a rule from this point: is it the result of $y = 2x$, or $y = x^2$, or pure noise? There is no way to tell. The point locks in information.

When data logic is coupled---for instance, when an agent's configuration data (model selection, temperature, role settings, scheduling rules) are scattered across code constants, environment variables, and database fields, all represented as the same kind of "setting strings"---the system loses the ability to independently operate on any single logical type. This is more than an engineering inconvenience. This is a **logical-type confusion**: heterogeneous categories share the same representational space, and their conditions of existence are bound together.

Data-logic coupling is the executive form of logical-type confusion at the data layer. **Strategic dimensional reduction**---the target of the Wall-Breaking Principle---is the executive form of logical-type confusion at the decision layer. They share the same structural error: **compressing orthogonal dimensions into the same representational space.**

### 0.2 The Liberation of the Function: Decoupling Domain and Expression

Decoupling $(x_1, y_1)$ into $f(x) = 2x$ is precisely executing **data-logic decoupling**.

$f$ is no longer a conflated point. It possesses an independent domain (what type of input it processes), an independent expression (how it maps from input to output), and an independent parameter space (it can be tuned independently without affecting other functions).

This is exactly the mathematical equivalent of the operation "identify compressed dimensions and establish independent decision layers" in the Wall-Breaking Principle. Data-logic decoupling is not a "preparatory step" for wall-breaking; **it is wall-breaking executed at the data layer**. Wall-breaking at the decision layer is the recursive application of the same operation at a higher level of abstraction.

This is the unified operational principle:

> **Principle of Logical-Type Decoupling**: No information-processing system should permit two categories of distinct logical types to share the same representational space. The unified repair operation is to identify compressed orthogonal dimensions, establish an independent representational layer (function) for each dimension, and build inter-layer communication through sequential interfaces.

### 0.3 Function as Update Unit: Bayesian Updating as the Formal Implementation of a General Update Operation

Every independent function that is decoupled out is an **update unit**. Its standard operation is: given an observation $x$, redistribute the state.

This is exactly the operational definition of Bayesian updating:

$$P(\theta \mid x) \propto P(x \mid \theta) \times P(\theta)$$

But note: no symbol in this formula is labeled "this is belief" or "this is cognition." What it describes is---there exists a parameter $\theta$, there exists an observation $x$, there exists a mapping rule from prior to posterior. $\theta$ can be a physical state, a model parameter, a role configuration, or a belief distribution---depending on what is executing the update.

Bayes' formula has never been labeled "belief updating." It describes the change rule, under evidence, of anything representable as a probability distribution. Cognition is merely a local case of this rule, not its essence. Therefore, the parameter tuning of an independent function shares the same operational kernel with Bayesian updating---given observation, redistribute state. Bayes' formula is the formal implementation of this kernel in probabilistic representation.

### 0.4 Nesting as Transition: How Function Nesting Produces Structural Change Between Spaces

When $f$ and $g$ nest as $g(f(x))$:

- The output of $f$ is the input of $g$. This means the observation space of $g$ is defined by the range of $f$.
- Updating $f$ changes the observation space of $g$. When the output interface of $f$ is modified, the legitimate input domain of $g$ undergoes structural change.

This distinguishes two kinds of updating:

- **Intra-space parameter updating**: tuning $\theta_f$ within the parameter space of $f$. This is the normal state---optimizing within a given dimensional space.
- **Inter-space structural transition**: modifying the output interface of $f$, thereby changing what $g$ can observe. This is wall-breaking---the hypothesis space of $g$ itself undergoes structural change.

Function nesting is not the serial connection of two "cognitive operations." It is two update units each executing an update, after which the upstream output becomes the downstream observation space. Dimension-free updating occurs when the output interface of $f$ is modified---this changes the legitimate input domain of $g$'s $P(x \mid \theta)$. This is not cognition changing cognition; it is things changing the observation conditions of things.

### 0.6 Formal Anchor: The General Form of the Update Operation

Before entering the terminology-unification table, we anchor the intuitive argument of "from point to function" as a formal definition. The function of this definition is to show that parametric Bayesian updating, multi-dimensional Bayesian updating, and function tuning in data-logic decoupling are not "analogies"---they are instantiations of the same formal operation on different object types.

> **Definition 0 (General Form of the Update Operation)**: For any object $X$, let its value space be $\mathcal{X}$, with distribution $P(X)$ defined thereon. Given observation $E$, the update operation maps $P(X)$ to $P(X \mid E) \propto P(E \mid X) \times P(X)$. This operation does not presuppose the semantic type of $X$.
>
> - When $X = h$ (hypothesis), it instantiates **parametric Bayesian updating**.
> - When $X = H$ (hypothesis space), it instantiates **multi-dimensional Bayesian updating (wall-breaking)**.
> - When $X$ is a data-configuration function, it instantiates **function tuning in data-logic decoupling**.

This definition is the foundation of the full-text formalization. It proves, at the operational level, that the mathematical structure of Bayes' formula does not care whether the object being updated is "belief," "hypothesis space," or "data-configuration function." Object types may differ; the operational form is strictly identical.

**An Honest Statement on the Scope of Definition 0**

Definition 0 subsumes function tuning in data-logic decoupling, parametric Bayesian updating, and multi-dimensional Bayesian updating under the same formal operation. The justification for this unification is that all three share the abstract operational structure "given input, map the current state of the object to a new state." Yet we must acknowledge two levels of asymmetry.

**First level: semantic distance between function tuning and probabilistic updating.** When an engineer splits conflated configuration data into independent functions and tunes their parameters, the inputs (observed system behavior) and outputs (modified function configuration) of this operation are not necessarily modeled as probability distributions. Calling this operation "Bayesian updating" in the strict sense requires us to endow the configuration state with probabilistic semantics---that is, to treat "the current configuration of the function" as a realization of a probability distribution. This requirement is not always met in current practice. Therefore, the word "updating" in Definition 0, when applied to non-probabilistic objects, should be understood as a **generalized structural isomorphism of operations**---given observation, redistribute state---rather than as a forced claim about its probabilistic semantics. Whether this generalized operational structure can, in all cases, be strictly embedded into the probabilistic semantics of the Bayesian framework is a question requiring subsequent formalization work; this paper leaves it open.

**Second level: symmetry between structural updating and parameter updating.** Definition 0 claims that multi-dimensional Bayesian updating (structural updating) shares the same form as parametric Bayesian updating. This symmetry holds at the level of mathematical expression---both are the product of prior and likelihood normalized to yield a posterior. But we must point out an operational asymmetry: in parameter updating, the prior $P(h)$ can usually be explicitly specified and computed within $H$; in structural updating, the computability of the structural prior $P(H)$ depends on our ability to construct a probability distribution over the higher-order object "hypothesis space." Section 4.2's Principle of Orthogonality Preservation provides a normative starting point for this, but transforming that principle into a computable estimator of $P(H)$ in actual systems requires independent algorithmic work. In other words, recursive structural correspondence (§2.5) holds in mathematical form, but its executability in engineering practice depends on the computabilization of the structural prior---a research question opened by the framework of this paper, not an unfinished item internal to it.

These two clarifications do not weaken Definition 0's function as the formal starting point of the full text. ## 1. Introduction: Updating Has a Hidden Dimension

Chapter 0 established the non-cognitive-binding premise of the update operation: "from point to function" is the root operation running through the data, decision, and structural layers. Bayesian updating is the formalization of this operation in the cognitive domain.

This chapter departs from the standard framework of Bayesian updating to reveal its implicit boundary.

Since Bayes' theorem was proposed in 1763, it has become the foundational framework for scientific inference, statistical decision-making, machine learning, and cognitive science [1--4]. Its core operation is concise and powerful:

$$P(h \mid E) = \frac{P(E \mid h) \times P(h)}{P(E)}$$

Given a hypothesis space $H$ (the set of all possible hypotheses) and a prior distribution $P(h)$ defined thereon, evidence $E$ maps the prior to the posterior $P(h \mid E)$---belief is revised in light of evidence.

This framework has achieved immense success over two and a half centuries, yet it has always presupposed an unexamined premise: **$H$ is given and fixed.** $h$ moves freely within $H$, but $H$ itself is outside the scope of updating.

This raises a question never formally posed:

> **When $H$ itself is insufficient to account for $E$---that is, no matter how $h$ is optimized within $H$, $P(E \mid h)$ remains persistently extremely low---how should updating proceed?**

Every empirical researcher has experienced this feeling: you have tuned countless parameters, swapped countless configurations, tried everything, yet accuracy is stuck right there and simply will not rise. This feeling, we call **wall-hitting experience**.

The existing Bayesian framework offers two responses to wall-hitting experience: (1) **Bayesian model selection**---using marginal likelihood to pick a better model from a pre-given candidate set $\{H_1, H_2, \ldots, H_k\}$ [5]; (2) **Bayesian model averaging**---weighting the posterior predictive distributions of multiple models [6]. But both presuppose that the set of candidate $H$'s is given in advance. They do not answer a more fundamental question: **How is a new $H$---a hypothesis space not yet in the candidate set---created?**

### 1.1 A Clue from the Wall-Breaking Principle

The Wall-Breaking Principle [7] provides a new perspective on this question.

That principle proves, in the specific domain of agent memory management, that any system compressing **Readiness** (can the data be directly consumed?) and **triggerability** (should the data be activated right now?)---two orthogonal decision dimensions---into a single scalar function necessarily commits overawarding or underemploying errors with probability 1 when environmental state switching requires reversed decision priority. In the most extreme conflation scenario, its discrimination accuracy degenerates to no more than 50%. This ceiling is called the **Dimensional Wall**---a structural boundary inevitably caused by dimensional compression and impossible to break through within the existing space.

The unique path to break through the Dimensional Wall is to execute **Dimensional Elevation of Strategy**: identify the compressed dimensions and establish an independent, sequentially executed decision layer for each. We call this operation **wall-breaking**.

Re-examined from the operational-ontological perspective of Chapter 0: strategic dimensional reduction is the manifestation of logical-type confusion at the decision layer---two orthogonal decision dimensions are compressed into the same scalar function. The Dimensional Wall is the inevitable cost of this confusion in decision accuracy. Wall-breaking is the repair operation of the Principle of Logical-Type Decoupling at the decision layer---expanding the conflated "point" into independent "functions." From the Bayesian statistical perspective, wall-hitting is parametric updating within an insufficient hypothesis space $H$, the Dimensional Wall is the likelihood limit of $H$, and wall-breaking is abandoning the old space $H$ and transitioning to a new space $H'$ capable of accommodating higher likelihood.

The Wall-Breaking Principle reveals to the Bayesian framework an operation never before formalized: **how to execute structural updating when parametric updating has reached its end.**

#### 1.2.1 The Three Layers of Meaning of Dimensional Freedom

A common misreading of "dimensional freedom" equates it with the one-directional "continuous ascension, abandoning the lower dimensions." We clarify explicitly here: dimensional freedom is not a one-directional stacking of dimensions, but a complete degree of freedom comprising three progressive layers. The essence of ascension is gaining the right to choose dimensions, not being permanently locked in a high-dimensional state.

1.  **Freedom of dimensional superposition, not freedom of replacement**
    Ascension is the sequential nesting and superposition of orthogonal dimensions, not the replacement of a low-dimensional space with a high-dimensional one. The lower dimensions always persist as the prerequisite input layer and foundational capability layer of the higher dimensions, just as building a skyscraper does not demolish the foundation. Every Dimensional Elevation of Strategy fully preserves all functions of the original dimensions; the new dimensions only expand the decision boundary on top of the original basis. There exists no operation of "abandoning the lower dimensions after ascension."

2.  **Freedom of dimensional scheduling, not freedom of locking**
    The core value of ascension is gaining the right to choose dimensional combinations: the system can freely switch the activated dimension set and adjust each dimension's decision priority according to environmental states and task requirements. Permanent locking in a single high-dimensional state is essentially still dimensional locking and has not gained complete dimensional freedom. Just as being able both to walk on land and to fly into the sky is true ascension, floating in the air forever without being able to land is merely switching to another locked dimension---compared with having both dimensions available, it is actually a capability reduction.

3.  **Freedom of scale compatibility, not freedom of one-directional scaling**
    The expansion of observational scale (e.g., adding microscopic observation capability to naked-eye macroscopic observation) is the unfolding of an independent orthogonal dimension---a standard ascension; bidirectional movement along the scale dimension (from macro to micro, or from micro to macro) is parameter translation within the new dimension, not dimensional reduction. Only the permanent loss of an independent information channel is dimensional reduction in the strict sense.

**Positioning Statement**: The above three layers of meaning are a clarification and completion of the core concept of "dimensional freedom"; they do not change the applicability conditions or falsification boundaries of this paper's core hypothesis, and serve only to eliminate the misreading of one-directional ascension and to clarify the complete semantics of "freedom."

## 2. Formal Narrative: Multi-Dimensional Bayesian Updating

This section provides a formal narrative in the language of Bayesian statistics for Dimension-Free Updatism. We first make clear: this is not "correcting" the Bayesian framework, but using the mathematical structure of Bayes' formula to describe an operation not previously formalized.

### 2.1 The Implicit Boundary of Standard Bayesian Updating

**Definition 1 (Standard Bayesian Updating)** Let $H$ be a hypothesis space, and $P(h)$ a prior distribution over $h \in H$. Given evidence $E$, the posterior distribution is:

$$P(h \mid E) = \frac{P(E \mid h) \times P(h)}{\int_{h' \in H} P(E \mid h') P(h') \, dh'}$$

In this definition, the boundary of $H$ is fixed. $h$ moves freely within $H$, but $H$ itself is not touched by $E$.

**Wall-hitting (boundary case)** When $\forall h \in H$, $P(E \mid h) \to 0$, the posterior distribution degenerates---$P(h \mid E)$ approaches a uniform distribution for all $h$. Belief collapses into an uninformative state.

At this point, the researcher experiences **wall-hitting experience**: no matter how parameters are adjusted, model performance cannot break through a certain invisible ceiling. The standard Bayesian framework diagnoses this as "model misspecification" and recommends "switching to a better model" [8]. But **how to produce that better model** lies outside the framework. The standard framework provides criteria for choosing among models, but does not provide the **operation of creating a new model**.

### 2.2 Formalization of Wall-Hitting Experience and Sequential-Test Criterion

**Definition 2 (Likelihood Sequence)** Let the system perform parametric Bayesian updating within hypothesis space $H$, producing a likelihood sequence:

$$\mathcal{L}_H = \{L_t\}_{t=1}^{T}, \quad L_t = \max_{h \in H} P(E_t \mid h)$$

where $E_t$ is the evidence observed at time $t$, and $L_t$ is the maximum likelihood of that evidence under $H$.

**Definition 3 (Wall-Hitting Experience)** Wall-hitting experience $E_{\text{wall}}$ is a decidable fact about the likelihood sequence $\mathcal{L}_H$, whose determination is formalized via Sequential Probability Ratio Testing (SPRT, Wald, 1945) [18].

Let two competing hypotheses be:

- $\mathcal{H}_0$: The current $H$ is sufficient---the observed low-likelihood sequence arises from random fluctuation. Under $\mathcal{H}_0$, the decline of $L_t$ should not exhibit a systematic trend.
- $\mathcal{H}_1$: The current $H$ is insufficient---what is observed is a structural collapse; the likelihood sequence has an insurmountable upper bound $U_H < L_{\text{req}}$ (the task-required likelihood threshold). This upper bound is the **Dimensional Wall**. Under $\mathcal{H}_1$, the optimal accuracy of the old space is structurally capped.

Set tolerances for two kinds of error:

- $\alpha = P(\text{reject } \mathcal{H}_0 \mid \mathcal{H}_0 \text{ true})$---probability of false wall-breaking (allergic to noise)
- $\beta = P(\text{accept } \mathcal{H}_0 \mid \mathcal{H}_1 \text{ true})$---probability of delaying necessary wall-breaking (trapped in a dead old space)

The sequential test updates the likelihood ratio after each new observation:

$$\Lambda_t = \frac{P(L_1, \ldots, L_t \mid \mathcal{H}_1)}{P(L_1, \ldots, L_t \mid \mathcal{H}_0)}$$

Decision rule:

- If $\Lambda_t \geq (1-\beta)/\alpha$: reject $\mathcal{H}_0$---$E_{\text{wall}}$ is confirmed, trigger wall-breaking (multi-dimensional Bayesian updating).
- If $\Lambda_t \leq \beta/(1-\alpha)$: accept $\mathcal{H}_0$---current fluctuation is judged noise; continue parametric updating within $H$.
- If $\beta/(1-\alpha) < \Lambda_t < (1-\beta)/\alpha$: insufficient evidence; continue observing.

The advantage of sequential testing is that it does not presuppose a fixed number of observations---the test terminates automatically when evidence is sufficient. In engineering practice this means: the system does not passively wait until it smashes into the Dimensional Wall before admitting the old space is dead; it actively monitors the statistical pattern of the likelihood sequence.

- **Meaning**: Existing parameter optimization can no longer bring the system to task requirements. $H_{\text{old}}$ is judged **inadequate**, but has not yet been proved **impossible**.

**Relation between the two levels**: The yellow alert is statistical evidence---it may be misjudged within the tolerance ranges of $\alpha$ and $\beta$. ### 2.4 Definition of Multi-Dimensional Bayesian Updating

**Definition 4 (Wall-Breaking---Multi-Dimensional Bayesian Updating)** Given wall-hitting experience $E_{\text{wall}}$, multi-dimensional Bayesian updating revises belief over the candidate space library $\mathcal{H}$:

$$P(H \mid E_{\text{wall}}) = \frac{P(E_{\text{wall}} \mid H) \times P(H)}{\sum_{H' \in \mathcal{H}} P(E_{\text{wall}} \mid H') P(H')}$$

where:

- $P(H)$ is the structural prior on $H$, whose formal foundation is given in §4.2.
- $P(E_{\text{wall}} \mid H)$ is the structural likelihood---the probability of observing wall-hitting experience under the premise that $H$ is sufficient.
- $P(H \mid E_{\text{wall}})$ is the structural posterior.

**Wall-breaking criterion**: When $P(H_{\text{current}} \mid E_{\text{wall}})$ drops below a preset belief threshold $\theta$---that is, the posterior belief in the sufficiency of the current $H$ is already extremely low---the system executes wall-breaking: abandoning $H_{\text{current}}$, and selecting a new $H_{\text{new}}$.

**Belief threshold $\theta$---setting principle**: $\theta$ should not be set independently of the two-level alarm system. A rational binding scheme is: calibrate $\theta$ to the structural posterior value of the system under red-alert trigger conditions. Since the red alert means the Dimensional Wall has been actually touched---that is, the insufficiency of $H_{\text{current}}$ is irreversibly proved---the structural posterior $P(H_{\text{current}} \mid E_{\text{wall}})$ at this point should approach zero. Therefore, $\theta$ should be set so that: in the yellow-alert interval (SPRT has crossed the threshold but the Dimensional Wall has not yet been touched), the system is in a **wall-breaking preparation state** rather than an **automatic wall-breaking state**---it should begin evaluating candidate replacement spaces, but not be forced to immediately execute the transition; true automatic wall-breaking occurs only after the red alert triggers. This division of labor prevents the system from prematurely abandoning a space that still has potential because of statistical fluctuations, and also prevents hesitation after the Dimensional Wall has been confirmed.

**Construction of the new space (Dimensional Elevation of Strategy)**: In $\mathcal{H}$, $H_{\text{new}}$ may not be a pre-given candidate. After wall-breaking is completed, parametric Bayesian updating restarts within $H_{\text{new}}$. The system resumes tracking convergent posteriors.

### 2.5 Recursive Isomorphism

**Proposition 1 (Recursive Structural Correspondence)** Parametric Bayesian updating and multi-dimensional Bayesian updating obey the same mathematical form:

- **Parametric updating**: $P_1(h \mid E) \propto P_1(E \mid h) \times P_1(h)$, operating on $h$ within $H$.
- **Structural updating**: $P_2(H \mid E_{\text{wall}}) \propto P_2(E_{\text{wall}} \mid H) \times P_2(H)$, operating on $H$ within $\mathcal{H}$.

The two are strictly identical in multiplicative structure and normalization structure---prior $\times$ likelihood $\to$ posterior. The only difference is the level of the object operated upon: parametric updating operates on hypotheses; structural updating operates on hypothesis spaces. Parametric updating is the projection of structural updating into a lower-dimensional space.

**Positioning Statement**: The "structural correspondence" here is a formal correspondence we observe within the Bayesian statistical framework. We do not claim it is a theorem in the strict sense, but present it as a narrative framework for understanding that parametric updating and structural updating share the same mathematical structure.

## 4. Dimensional Enhancement of Bayesian Statistics

### 4.1 Formal Extension of the Domain of Bayesian Updating

The domain of traditional Bayesian updating is:
$$\{(\text{prior}, \text{evidence}) \to \text{posterior} \mid \text{all within given } H\}$$

Multi-dimensional Bayesian updating extends the domain to:
$$\{(\text{prior}, \text{evidence}) \to \text{posterior} \mid \text{within } H\} \;\cup\; \{(\text{structural prior}, \text{wall-hitting experience}) \to \text{structural posterior} \mid \text{within } \mathcal{H}\}$$

And the two operations are unified by recursive structural correspondence: parametric updating is the projection of structural updating into $H$; structural updating is the projection of parametric updating into $\mathcal{H}$.

This extension provides, for the first time, a formal dynamics for the question "where does the hypothesis space come from?": it is the posterior of multi-dimensional Bayesian updating---after belief in the old space collapses, the system redistributes probability mass over the candidate space library, and constructs a new space via Dimensional Elevation of Strategy.

### 4.2 Principle of Orthogonality Preservation: The Normative Foundation of the Structural Prior $P(H)$

**Definition 5 (Principle of Orthogonality Preservation)** Before observing any wall-hitting experience, a rational Bayesian agent's structural prior belief $P(H)$ over hypothesis space $H$ should be proportional to the number of orthogonal decision-information channels preserved by $H$, and inversely proportional to the number of conflated orthogonal dimension pairs compressed by $H$.

This principle is not an externally injected philosophical preference, but a corollary based on the Wall-Breaking Principle: any system compressing orthogonal dimensions is locked in by the Dimensional Wall. Therefore, before observing specific evidence, a rational agent should assign higher structural prior to architectures that "preserve orthogonal information channels"---not because they are "simpler," but because such architectures are not logically preset with an insurmountable performance ceiling.

Formally:
$$P(H) \propto \exp(-\lambda \cdot \text{CompressedDims}(H))$$

where $\text{CompressedDims}(H)$ is the number of conflated orthogonal dimension pairs in $H$, and $\lambda > 0$ is the intensity parameter of orthogonality preservation.

**Positioning Statement**: This principle is a normative recommendation based on the Wall-Breaking Principle, providing a rational foundation for the structural prior within the framework of that principle. It is not a necessary conclusion deduced from any theorem, but a prudent choice based on the operational principle of "from point to function."

**Relation between the Principle of Orthogonality Preservation and standard Occam's razor**: In standard Bayesian model selection, Occam's razor prefers models with fewer parameters---because simpler models have more concentrated prior predictive distributions [9]. The Principle of Orthogonality Preservation operates at a different layer from this standard principle and does not constitute direct competition. Occam's razor is a prior preference at the parameter layer (within $H$): given a hypothesis space, prefer simpler hypotheses within it. The Principle of Orthogonality Preservation is a prior preference at the structural layer (over $\mathcal{H}$): among multiple hypothesis spaces, prefer those that preserve orthogonal information channels. The two constitute a sequential relationship---first use the Principle of Orthogonality Preservation to select or construct an $H$ in the candidate space library, then use standard Bayesian methods to apply Occam's razor to $h$ within that $H$. A decoupled architecture may have more parameters (disadvantageous for Occam's razor at the parameter layer), but better orthogonality preservation (advantageous at the structural layer); at this point the Principle of Orthogonality Preservation does not "overturn" Occam's razor---it simply provides a logic at the structural selection stage that differs from the parameter-layer logic. The two principles fulfill different hierarchical functions in the same modeling decision.

### 5.2 Integrating Multi-Dimensional Bayesian Updating into the Variational Free-Energy Framework of Active Inference

Active inference [4] is one of the cornerstone frameworks of modern cognitive science. Its core quantity is variational free energy $\mathcal{F}$---the negative log upper bound of model evidence:
$$\mathcal{F} = D_{KL}[Q(s) \,\|\, P(s \mid o)] - \ln P(o \mid \text{model})$$
where $Q(s)$ is the agent's approximate posterior over world state $s$, $P(s \mid o)$ is the true posterior given observation $o$ and current internal model, and $P(o \mid \text{model})$ is model evidence.

The core operation of active inference is: the agent continuously lowers free energy through perception (updating $Q(s)$ to minimize $\mathcal{F}$) and action (changing $o$ to minimize $\mathcal{F}$).

**Wall-breaking as recursive extension of active inference**: When the agent, after optimizing beliefs and actions, still cannot make $\mathcal{F}$ converge to a tolerable level---that is, model evidence $P(o \mid \text{model})$ remains persistently extremely low, and the agent experiences wall-hitting---the standard active-inference framework specifies no further operation rules.

Multi-dimensional Bayesian updating intervenes here: putting the model itself $M$ into a candidate space library $\mathcal{M}$, and defining a meta-version of variational free energy $\mathcal{F}_{\text{wall}}$:
$$\mathcal{F}_{\text{wall}} = D_{KL}[Q(M) \,\|\, P(M \mid E_{\text{wall}})] - \ln P(E_{\text{wall}})$$
where $Q(M)$ is the agent's approximate posterior over model $M$.

Wall-breaking is the recursive application of active inference at the model level---not minimizing free energy $\mathcal{F}$ about state $s$, but minimizing free energy $\mathcal{F}_{\text{wall}}$ about model $M$.

**Three-level unified updating agent**: Multi-dimensional Bayesian updating thereby completes a three-level unified theory of the updating agent:

- **Perception**: parametric Bayesian updating---given model, update state belief, minimize $\mathcal{F}$.
- **Action**: active inference---given model, change input through action to match prediction, minimize $\mathcal{F}$.
- **Wall-breaking**: multi-dimensional Bayesian updating---given wall-hitting experience, update model itself, minimize $\mathcal{F}_{\text{wall}}$.

These three levels share the same core---minimizing variational free energy. The only difference is the level of the object operated upon: state, action sequence, model. When the agent executes wall-breaking---optimizing the model itself---it is not executing a "higher-level cognitive reasoning," but executing the same ontological updating operation as perception and action, with the object of operation shifted from state to model.

#### 6.3.6 An Operational Reformulation of Hume's Problem of Induction: From "Defense" to "Diagnosis"
