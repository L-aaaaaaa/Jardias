# 破壁原理：论认知架构中维度混叠的结构性脆弱性

## ——一项可证伪假说与操作指南


**摘要**

本文提出一个可证伪的假说——**破壁原理**——用以诊断和解释长程语言智能体中一类特定的记忆管理结构性脆弱性。核心主张是：任何将质量(Q)与相关性(R)这两个逻辑独立的判断维度压缩进单一标量函数的系统（我们称之为**维度混叠（Dimensional Conflation）**），在环境状态切换要求反转决策优先序时，必然在至少一种状态下以概率1犯下**过奖（Overaward）**或**屈才（Underemploy）**错误；该错误无法通过参数调优消除。我们首先从上下文窗口有限这一唯一的硬性约束出发，推导出记忆管理的二元决策结构，并引入序贯概率比检验为碰壁经验提供可操作的判定方法。随后，我们在理想化模型中严格证明了假说的逻辑内核，并在受控实验中验证了其核心预测：固定线性排序在环境切换时达到100%错误，而一种将Q与R解耦为序贯决策层的**策略升维（Dimensional Elevation of Strategy）**架构将此错误降为零（McNemar检验 p=6.1e-5）。基于此，本文提出一项元认知决策规则：当黄色警报（序贯检验确认结构不充分）触发时，继续参数优化的边际收益已开始崩溃，策略升维的预期收益显著更高——因此在两个方向都具备成熟实施条件的前提下，资源应向策略升维方向倾斜；当红色警报（次元壁50%退化点）触及时，破壁不再是选项，而是逻辑必然。我们进一步划定了假说的理论边界并给出了明确的证伪条件。跨领域启发性延伸被明确标记为推测性类比。本文不宣称发现普遍适用的自然定律，而是试图为一种被工程直觉感知但尚未被严格分析的脆弱性，提供第一个可被检验的形式化框架、配套的概念工具，以及一项可跨领域应用的理性决策规则。

## 1. 引言：一个工程直觉及其文明尺度的意义

### 1.1 反常现象的观察

在构建长程语言智能体的工程实践中，我们反复观察到一种难以归因的失败模式。一个在大多数情况下表现优异的记忆管理系统，在任务进行到后期、上下文窗口被大量中间结果填满时，有时会做出一个明显错误的选择：它将高分授予了一条格式混乱、无法被模型直接解析的原始系统日志，却将一个格式清晰、语义自足的经验总结排除在了注意力窗口之外。

事后分析表明，这两条数据的关键差异在于：原始日志虽然与当前调试目标高度相关，但处于“不可直接消费”的状态；而经验总结虽然格式完美，却服务于一个已经完成的子任务，与当前目标不再相关。系统用一个单一的语义相似度分数来排序这两条数据，而在这个分数中，上述两个维度的信息被不可逆地混叠了。

我们将这两种错误分别命名为：

- **过奖**：系统将注意力颁给一条尚未就绪的数据。它正确判断了数据的相关性，但错误判断了其可消费性。
- **屈才**：系统将一条已就绪的数据闲置。它正确判断了数据的可消费性，但错误判断了其当下的相关性。

### 1.2 这不是一个新问题——这是一个被反复支付的历史代价

“过奖”和“屈才”虽然是在Agent记忆管理领域被命名的，但它们所指向的结构性困境，在人类文明史中处处可见：

- **天文学**：托勒密体系在1400年间不断增加本轮均轮——在旧空间内反复参数优化——直到哥白尼的日心说执行了一次策略升维，才终结了这场漫长的伪优化。
- **医学**：放血疗法在两千多年里不断“优化放血时机、部位、血量”，直到病原说的范式革命才将其淘汰。每一次调整放血方案，都是在错误空间内的一次精进。
- **软件开发**：无数项目在单体架构内反复优化——缓存、分库分表、异步化——最终不得不重构为微服务架构。每一次延迟的架构重构，都意味着大量工程师时间被浪费在注定触顶的优化上。

这些案例共享同一个模式：在错误的假设空间内过度优化，因为缺乏“何时切换”的判据。

现在，请允许我们暂时停下论证，做一个小小的计算。

如果托勒密天文学提前九百年被识别为“碰壁”——不是因为哥白尼的天才偶然降临，而是因为天文学家拥有一个基于证据的判据，能够区分“本轮精度不足”和“本轮范式本身已无改进空间”——人类科学革命会提前多久？

如果放血疗法提前一千五百年被终止，不是因为巴斯德的偶然发现，而是因为医学界拥有一个统计工具，能够在体液说的似然序列持续崩塌时发出警报？

这些“如果”当然不是历史。但它们指向一个可以精确陈述的事实：在这两种情况下，人类缺少的不是新理论，而是“何时该放弃旧空间”的判据。我们在旧框架内投入了额外的几个世纪——不是因为最优策略需要这么多时间，而是因为我们在一个无法突破的结构上限面前，继续调了一千年的参数。

## 2. 概念框架与形式化

### 2.1 从硬约束推导出两个不可归约的决策问题

考虑一个在任何时刻都面临如下硬性约束的语言智能体：

> **上下文窗口有限。** 模型在单次推理中能有效处理的信息量存在严格上限。

从这个唯一的硬性约束出发，任何记忆管理策略都必须回答两个不可归约的问题：

**问题一（质量判断）**：在无限的外部信息和累积的历史数据中，哪些条目具备被纳入上下文窗口的**基本条件**？这涉及数据的体积是否超出窗口容量、格式是否能被模型原生解析、是否具有语义自足性（即在无外部参照物的情况下可被独立理解）。

**问题二（相关性判断）**：在通过了基本条件筛选的候选集中，哪些条目**此刻**应该被实际触发进入上下文窗口？这涉及数据是否服务于当前正在执行的任务、是否已退出工作集、是否对当前推理步骤具有不可替代的信息价值。

这两个问题在逻辑上是不可互归的。前者关心数据的**内在属性**（“能不能”被消费），后者关心数据的**关系属性**（“该不该”此刻被触发）。前者是后者的前提——只有先确定了候选集，才能在其中进行选择——但前者的答案不能决定后者的答案。一条数据可以具备完美的内在属性（体积小、格式清晰、语义自足），却完全不服务于当前目标。反之，一条对当前目标至关重要的数据，可能因为处于未加工状态而不具备被直接触发消费的条件。

因此，任何记忆管理策略，都必然隐含地执行这两个决策步骤。即使一个系统使用端到端学习的策略，不显式地命名“质量”和“相关性”这两个范畴，其损失函数在优化过程中也必然要学习近似这两个维度的决策——因为物理层面的上下文窗口约束，会迫使任何有效的策略最终区分“能放进去的信息”和“该放进去的信息”。

**声明**：将这两个问题识别为“不可归约”的，是我们基于上下文窗口这一特定硬约束所做出的建模选择。我们承认，在更复杂的认知架构中，可能存在其他同样重要的决策维度（例如，若引入“信息必须真实”这一独立的硬性约束，将导出“可信度”维度）。本文的框架聚焦于Q和R这两个维度的混叠及其后果，并欢迎有研究者提出更全面的竞争性模型。

> *质量问“能不能用”，相关性问“该不该用”——混在一起，就分不清谁因何而得分。*

### 2.2 形式化定义

**定义1（质量Q与相关性R）** 令 $\mathcal{D}$ 为所有可能进入上下文窗口的数据集合。

- **质量** $Q(d) \in \{0, 1\}$：$Q(d)=1$ 当且仅当数据 $d$ 同时满足：(1) 体积不超过上下文窗口限制；(2) 格式可被当前模型原生解析；(3) 语义自足——$d$ 中所有指代、省略和隐含前提均已显式化，模型可无需访问 $d$ 之外的任何上下文即可完整理解其含义。
- **相关性** $R_t(d) \in \{0, 1\}$：$R_t(d)=1$ 当且仅当数据 $d$ 的内容包含对当前任务目标 $g_t$ 有用的信息。

**定义2（维度混叠系统）** 设系统使用单一标量函数 $h: \mathcal{D} \to \mathbb{R}$ 为每条数据打分，并仅依据 $h$ 的分数排序来选择注入上下文窗口的数据。若 $h$ 的定义域中不显式包含当前环境状态 $S$ 作为独立可操作的输入通道——即 $h$ 对于给定的数据，其输出不随环境状态变化而改变——则称该系统处于**维度混叠**状态。

我们使用“维度混叠”一词，因为它精确地描述了问题的本质：两个独立维度的信息在通过 $h$ 的瓶颈时发生了不可逆的混合，而非泛指任何形式的策略简化。

### 2.3 过奖与屈才：两类系统性错误

维度混叠导致了两类相互对称的认知错误：

**过奖（假阳性能力错配）**：系统错误地将注意力颁给尚未就绪的数据。一条格式混乱的原始系统日志，因为与当前调试目标语义相似度高，被直接注入上下文窗口——它挤占了有效的认知空间。系统正确判断了它的相关性价值，但错误判断了它的质量状态。在形式上，系统选择了一条 $Q=0, R=1$ 的数据。

**屈才（假阴性能力错配）**：系统错误地将已就绪的数据闲置。一条已精炼的经验总结，格式清晰、语义自足，但因为与当前局部查询的文本相似度不高，被排除在上下文窗口之外。系统正确判断了它的质量状态，但错误判断了它的相关性价值。在形式上，系统选择了一条 $Q=1, R=0$ 的数据。

以下定理穷举了记忆管理失败的所有可能类型。此定理为定义穷举，独立于后文假说的成立与否。

**定理1（错误穷举定理）** 若一个系统的上下文窗口 $C_t$ 在时刻 $t$ 未能包含最优信息组合 $C^*$，则该失败必然可以归因于质量函数的至少一个错误（能力错配）或相关性函数的至少一个错误（意图失焦）。四种错误类型——假阳性能力错配（过奖）、假阴性能力错配（屈才）、假阳性意图失焦、假阴性意图失焦——覆盖了 $C_t \neq C^*$ 的所有可能。□

### 2.4 攻击面界定：谁落入这个框架

在界定攻击面之前，我们必须首先澄清维度混叠在工程实践中最常见的三种实现形式：

1. **严格单标量**：系统字面上使用一个标量分数进行排序，如纯语义相似度。
2. **加权聚合**：系统使用多个信号（如语义相似度、时间新近度、重要性标记）的加权和，但最终压平为一个分数——这不构成独立的序贯决策层。
3. **隐式混叠**：函数的输出是一个标量，但其内部计算可能在高维空间中隐式地融合了Q和R的信息（如神经网络embedding的余弦相似度）。只要最终输出是用于排序的单一标量，多维度到单一序的映射必然产生不可区分的等价类。

任何不包含独立的、序贯的、先于相关性选择的质量审查层的架构，其最终决策都必然落入以上三者之一。

基于各框架公开发布的默认行为，以下主流方案均落入维度混叠的范畴：

- **LangChain ConversationBufferMemory**：对话历史按时间顺序追加，超出token上限时按FIFO截断。无独立质量判断——所有数据被视为同质。
- **MemGPT / Letta**：虚拟内存分页，页面置换策略依赖时间局部性，不区分质量和相关性。
- **CrewAI**：语义相似度（相关性的代理）和元数据过滤（质量的粗糙代理）被压平在同一个排序函数中。
- **AutoGen**：对话历史全量传递，无任何记忆管理策略。

这些系统使用的信号数量和类型各不相同，但它们共享同一个结构特征：**没有一个独立的、先于相关性选择执行的质量审查层。** 颁奖机制——即注意力分配函数——同时承担了审查获奖资格和决定获奖对象的双重职能。

**模型切换测试**：我们提出一个可操作的检验标准，用于判断一个系统是否落入维度混叠的范畴。将底层模型切换为上下文窗口大小、解析能力显著不同的新模型。若系统在切换后没有任何独立环节重新审查数据的质量（“这条数据在新模型下还能被直接消费吗？”），则处于维度混叠状态。

值得指出的是，某些RAG管线在实践中已经初步触碰到了解耦的边界：它们在检索阶段之后增加了一个独立的重排序或过滤阶段，这可能包含对数据质量的隐式检查。这些实践者的工程直觉，正是本文试图形式化的对象。关于维度混叠为何在主流框架中如此普遍，见附录C的解耦引理。


## 3. 假说的逻辑内核与形式化证明

### 3.1 假说的精确陈述

> **破壁原理（可证伪假说版本）**：对于处于维度混叠状态的记忆管理系统，在环境状态发生切换、要求系统反转其决策优先序的条件下：
>
> (1) **固定排序的维度错位**：系统的固定排序必然在至少一种状态下以概率1犯下过奖或屈才错误。
>
> (2) **不可消除性**：该错误的根源在于固定映射的定义域不包含环境状态。参数调优只能改变错误发生的具体位置（在哪个状态下犯错），不能消除错误发生的必然性。
>
> (3) **信息混叠退化**：在最极端的混叠情形下，当单标量函数将Q与R的差异压缩至完全相同的分数时（$h(d_A) = h(d_B)$），系统对决定性维度的信息完全丧失，区分准确率退化至不高于50%。此退化极限被称为**次元壁**。

> *宽度量尽难知高，铁鞋踏破不成飞。*

### 3.2 理想化模型设定

为在受控条件下严格检验假说的逻辑内核，我们构造一个理想化的任务模型。**我们公开声明：这是一个被故意简化到极端的人造环境。它的功能不是模拟现实，而是作为假说最尖锐的测试条件。**

**数据**：两个文档——

- $d_A$：高质量 (Q=1)、低相关性 (R=0)
- $d_B$：低质量 (Q=0)、高相关性 (R=1)

**环境状态**：

- $S_{\text{hungry}}$（信息饥渴）：上下文窗口充裕，允许二次解析。低质量数据可被加工后消费，正确选择是 $d_B$（优先相关性，以获取更多任务相关信息）。
- $S_{\text{scarce}}$（资源紧缺）：上下文窗口接近饱和，无额外计算步骤可用于二次解析。注入低质量数据将直接导致解析失败，正确选择是 $d_A$（优先质量，以确保至少能获得部分有效推理）。

**上下文窗口容量**固定为 1，即每步只能选择一个文档。两种状态的正确选择互斥——这正是假说所预言的“反转优先序”的最严格测试条件。

### 3.3 引理1：固定排序的维度错位风险

**引理1（固定排序的维度错位风险）** 设 $h$ 为维度混叠系统的固定排序函数，其输出不随环境状态改变。则在 $S_{\text{hungry}}$ 和 $S_{\text{scarce}}$ 交替出现的任务中，系统必然在至少一种状态下以概率1犯错。该错误无法通过参数调优消除。

*证明*：$h$ 是从数据表示到标量的固定映射。其输出不随环境状态改变。因此 $h(d_A)$ 与 $h(d_B)$ 的大小关系是全局固定的。

**情形1**：$h(d_A) > h(d_B)$。系统在任何状态下都优先选 $d_A$。在 $S_{\text{scarce}}$ 下正确，在 $S_{\text{hungry}}$ 下以概率1犯错（应选 $d_B$）。

**情形2**：$h(d_B) > h(d_A)$。系统在任何状态下都优先选 $d_B$。在 $S_{\text{hungry}}$ 下正确，在 $S_{\text{scarce}}$ 下以概率1犯错（应选 $d_A$）。

**情形3**：$h(d_A) = h(d_B)$。退化至随机选择（见引理2）。

正确排序在两种状态下恰好互斥：$\rho^*(S_{\text{scarce}}) = (d_A, d_B)$，$\rho^*(S_{\text{hungry}}) = (d_B, d_A)$。不存在一个全局固定排序能同时满足两者。参数调优只能改变 $h$ 的权重，从而改变错误发生在哪个状态下，不能消除错误发生的必然性——因为固定映射本身无法感知环境状态的切换。□

> *铁鞋踏破不成飞。一个不随环境改变的系统，在环境要求改变时必然失败。这不是工程参数的失调，而是架构选择的逻辑必然。*

### 3.4 引理2：信息混叠退化（次元壁）

**引理2（信息混叠退化）** 当 $h(d_A) = h(d_B)$ 时，系统对决定性维度的信息完全丧失。在此退化条件下，系统的区分期望准确率不高于50%。

*证明*：当 $h(d_A) = h(d_B)$ 时，系统从 $h$ 的输出中无法获得任何关于 $d_A$ 和 $d_B$ 在Q维度或R维度上独立取值的信息。正确决策的维度 $Y$ 在 $S_{\text{scarce}}$ 下为 $Q$（应选高质量者），在 $S_{\text{hungry}}$ 下为 $R$（应选高相关性者）。系统对 $Y$ 的信息完全缺失。在系统视角下，两条数据在 $Y$ 维度上的排序是等可能的。因此期望正确率不高于50%。□

*注：引理2所依赖的决策论约束——当决策者对决定性维度的信息完全缺失时，期望正确率不高于50%——在贝叶斯框架中对应无信息先验下的对称后验，在频率学派框架中对应最坏情况下的极小化极大策略，在信息论框架中对应零互信息信道的容量退化。因此，此约束不是某个特定统计流派的内部假设，而是横跨三大框架的、不可绕过的决策论限制。*

> *宽度量尽难知高。当两个点在投影中重合时，区分它们的任何尝试都退化为随机猜测。*

**次元壁的极简几何直觉**

平面上有两个点A和B。它们的x坐标相同，y值未知。

现在要判断：谁在上？

正确答案依赖于y值。但y值不可见。你只能猜。任何策略的期望正确率，不可能超过50%。

这就是次元壁的全部秘密。50%不是工程困难——是零信息输入下，区分两个不可区分对象的理论上限。

**关键限定**：50%是本理想化模型中最极端混叠情形的理论退化点，其功能是确立一个基准参考水平。我们不宣称50%是维度混叠系统在一般情形下的普适准确率上限。当 $h(d_A) \neq h(d_B)$ 时，系统的平均准确率可能远高于50%。假说的核心攻击点不是平均性能，而是固定排序在环境切换时确定性失败的必然性。

### 3.5 主假说的形式化陈述

综合引理1和引理2，假说的形式化陈述如下：

对于任何处于维度混叠状态的记忆管理系统——即使用单标量函数 $h: \mathcal{D} \to \mathbb{R}$ 联合管理Q与R，且 $h$ 的定义域不包括环境状态 $S$：

(1) 存在一个非空的环境状态集合 $\mathcal{S}_{\text{err}} \subseteq \{S_{\text{scarce}}, S_{\text{hungry}}\}$，使得当系统处于 $\mathcal{S}_{\text{err}}$ 中的状态时，系统以概率1做出错误决策。

(2) 该风险的根源在于 $h$ 的定义域不包括环境状态 $S$。参数调优只能改变 $h(d_A)$ 与 $h(d_B)$ 的相对大小，从而改变 $\mathcal{S}_{\text{err}}$ 的具体元素，但不能使 $\mathcal{S}_{\text{err}}$ 为空集。

(3) 当 $h(d_A) = h(d_B)$ 时，$\mathcal{S}_{\text{err}} = \{S_{\text{scarce}}, S_{\text{hungry}}\}$，期望准确率退化至不高于50%。

(4) 使 $\mathcal{S}_{\text{err}}$ 为空的唯一方法，是引入以环境状态 $S$ 为输入的条件化机制，即用 $h_S: \mathcal{D} \times \mathcal{S} \to \mathbb{R}$ 替代 $h: \mathcal{D} \to \mathbb{R}$。这在架构上等价于策略升维——将质量判断与相关性判断解耦为两个可独立条件化的决策层。

> *固定排序在环境切换时必然犯错——不是参数问题，是架构问题。*


## 4. 受控实验验证

我们设计受控实验来验证假说的核心预测：固定排序系统在环境切换时是否表现出确定性失败，以及解耦操作是否能消除这一失败。

### 4.1 实验设计

**任务结构**：30个任务，前15个为 $S_{\text{hungry}}$ 状态，后15个为 $S_{\text{scarce}}$ 状态。每个任务包含两个文档：$d_A$ (Q=1, R=0) 和 $d_B$ (Q=0, R=1)。Q和R取值为极值{0,1}，构成最严苛的区分条件。

**对比策略**：

1. 随机基线：均匀随机选择
2. 固定线性 (Q=0.0, R=1.0)：仅按相关性排序
3. 固定线性 (Q=1.0, R=0.0)：仅按质量排序
4. 固定线性 (Q=0.5, R=0.5)：等权混合
5. 最佳固定线性：通过网格搜索（步长0.1）找到的使总错误最少的一组权重
6. 策略升维：环境感知选择——$S_{\text{hungry}}$ 下按R排序，$S_{\text{scarce}}$ 下按Q排序

**上下文窗口容量**固定为1。两种状态的正确选择互斥。

**统计检验**：McNemar检验比较最佳固定线性与策略升维在 $S_{\text{scarce}}$ 状态下的成对正确性。

### 4.2 实验结果

| 策略                        | Hungry错误率 | Scarce错误率 |
| :-------------------------- | :----------- | :----------- |
| 随机基线                    | 60.0%        | 26.7%        |
| 固定线性 (0.0Q, 1.0R)       | 0.0%         | 100.0%       |
| 固定线性 (1.0Q, 0.0R)       | 100.0%       | 0.0%         |
| 固定线性 (0.5Q, 0.5R)       | 0.0%         | 100.0%       |
| 最佳固定线性 (Q=0.0, R=1.0) | 0.0%         | 100.0%       |
| **策略升维**                | **0.0%**     | **0.0%**     |

网格搜索得到的最佳固定线性权重为 (Q=0.0, R=1.0)，其总错误为 15/30。该策略在 $S_{\text{hungry}}$ 下完全正确，但在 $S_{\text{scarce}}$ 下**100%错误**。任何其他固定线性组合（包括Q=1.0, R=0.0）都在另一状态达到100%错误。策略升维在两种状态下均达到**0%错误**。

**McNemar检验**：比较最佳固定线性与策略升维在 $S_{\text{scarce}}$ 状态下的成对正确性，p-value = 6.1e-5 < 0.05，差异统计显著。

### 4.3 实验解释

1. **固定排序的确定性失败被验证**：无论选择哪一组固定权重，总有一个环境状态达到100%错误。即使通过网格搜索选取“最佳”权重，也无法避免尾部确定性风险——最佳权重将全部正确性押注在Hungry状态，导致Scarce状态完全失效。这与引理1的预测完全一致。

2. **策略升维消除该失败模式**：显式引入环境状态感知通道，根据当前状态选择决策维度（Hungry用R，Scarce用Q），在两个状态均获得零错误。

3. **实验的功能定位**：此实验在极度简化的人造环境中进行，验证的是脆弱性的**存在性**，而非其**普遍性**。现实任务的复杂性——高维嵌入、噪声、Q-R统计耦合——可能使脆弱性的表现形式更加隐蔽，但不会改变其结构根源。我们欢迎后续工作在更真实的数据集和任务上进行独立复现和扩展验证。

## 5. 决策规则：参数优化与策略升维的优先级判据

假说的实践价值，不仅在于诊断脆弱性，更在于它揭示了一个此前未被形式化的元认知决策问题：**在“继续参数优化”和“执行策略升维”这两个都具备成熟实施条件的选项之间，如何基于证据做出理性选择？**

现实中，绝大多数系统和团队的默认选择是前者——继续优化。原因包括路径依赖、风险厌恶、短期可见的边际收益，以及最关键的一点：缺乏“何时停止优化”的统计判据。策略升维往往被推迟到“实在走投无路”时才被考虑——而那时已经浪费了大量资源。

本节基于假说的逻辑内核，提出一套基于序贯概率比检验的决策规则，为上述元认知决策问题提供可操作的判据。

### 5.1 黄色警报：结构不充分预警

**定义3（似然序列）** 设系统在假设空间 $H$ 内进行参数优化，产生似然序列：

$$\mathcal{L}_H = \{L_t\}_{t=1}^{T}, \quad L_t = \max_{h \in H} P(E_t|h)$$

其中 $E_t$ 为第 $t$ 次观测到的任务表现，$L_t$ 为该表现下 $H$ 内的最优似然。

**黄色警报的触发**：使用序贯概率比检验（SPRT, Wald, 1945）监测似然序列。设两个竞争假设：

- $\mathcal{H}_0$：当前 $H$ 是充分的——观测到的低似然来自随机波动。
- $\mathcal{H}_1$：当前 $H$ 不充分——存在一个不可突破的结构上限 $U_H < L_{\text{req}}$（任务要求的似然阈值）。

设定两类错误的容忍度：

- $\alpha = P(\text{拒绝}\mathcal{H}_0 \mid \mathcal{H}_0\text{真})$——错误破壁的概率（对噪声过敏）
- $\beta = P(\text{接受}\mathcal{H}_0 \mid \mathcal{H}_1\text{真})$——延迟必要破壁的概率（困在已死的旧空间中）

序贯似然比 $\Lambda_t$ 在每次新观测后更新。当 $\Lambda_t \geq (1-\beta)/\alpha$ 时，拒绝 $\mathcal{H}_0$——黄色警报触发。

**黄色警报的含义**：系统已确认“当前 $H$ 的结构上限低于任务要求”。继续在 $H$ 内进行参数优化的边际收益已开始崩溃。系统应进入**破壁准备状态**——开始评估候选替代假设空间，或准备执行策略升维。

**补充判据**：当黄色警报触发时，除监测似然序列外，还可检视错误本身的模式。若系统在特定环境状态下同时且稳定地犯下过奖与屈才两类错误——即“该进的被挡，不该进的被放”——这种对称性失败本身就是维度混叠的强烈信号，它提示系统中可能存在两个尚未独立的判断通道。

### 5.2 红色警报：次元壁终局性确认

**红色警报的触发**：当系统在 $H$ 内的最优准确率触及50%（或在其他维度压缩结构下对应的理论上限）——即 $H$ 的结构上限 $U_H$ 被实际观测到。

**红色警报的含义**：$H$ 的不充分性被不可逆地证明。50%不是最坏情况警告，而是该空间内的理论极限。破壁不再是选项——它是逻辑必然。

### 5.3 决策规则的操作总结

| 状态     | 判据                                 | 决策                                    |
| :------- | :----------------------------------- | :-------------------------------------- |
| 正常     | 似然稳定高于 $L_{\text{req}}$        | 继续在 $H$ 内进行参数优化               |
| 黄色警报 | 序贯检验跨过 $(1-\beta)/\alpha$ 阈值 | 评估替代 $H$，准备破壁                  |
| 红色警报 | 准确率触及 $H$ 的理论退化点          | 执行破壁，不再在 $H$ 内投入任何优化资源 |

> *黄灯亮起时，优化已接近触顶。红灯亮起时，换空间是唯一出路。*

### 5.4 优先级倾斜：碰壁之后的资源分配规则

黄色警报的意义，不仅是告诉系统“该停了”。它同时告诉系统另一件事：在旧空间内继续投入资源的边际收益已经崩溃，而升维的预期收益——至少能打破次元壁——在统计上是正的。

这不是说升维一定成功。升维可能失败——新空间可能选错，解耦操作可能设计不当，新架构可能不如旧架构。但黄色警报确认了一件事：留在旧空间内，收益上限已被锁死。而此时升维的期望——即使将失败概率计入——仍然高于继续优化的期望，因为继续优化的期望正在趋近于零。

因此，一个理性的资源分配规则是：**当黄色警报触发时，在“参数优化”与“策略升维”两个方向都具备成熟实施条件的前提下，资源应向策略升维方向倾斜。** 这不是因为升维更“正确”，而是因为在旧空间的边际收益已低于任何可能打破次元壁的尝试的预期收益。

一个历史注记可以说明这个规则的力量。在哥白尼之前，天文学家在托勒密框架内优化本轮均轮已持续一千四百年。如果当时存在一个基于证据的判据，能够在本轮优化的边际精度增益开始崩溃时发出警报，并将资源从“增加更多本轮”转向“寻找替代几何结构”——那么，科学革命可能不是始于十六世纪的一次天才偶然，而是始于一个可以被统计检验触发的、制度性的资源再分配决策。

我们并不宣称这个规则在所有条件下最优。我们只宣称：在本文严格界定的边界内——即两个方向都具备实施条件，且黄色警报已触发——优先向升维倾斜是理性的。如果升维的可行性未知（例如，新空间尚未被构想出来），规则的任务不是强制冒险，而是**停止在旧空间内的无效优化，并将资源投入探索**。探索可能失败。但继续优化已确定会失败。在确定失败和可能失败之间，规则的选择是清晰的。

**直觉的前统计地位**

本文以序贯概率比检验为黄色警报提供了形式化判据，但承认在实践中，熟练的实践者常在数据达到统计显著性之前，就感知到“此路不通”的直觉。我们不将直觉神秘化，而是将其理解为：人脑作为模式识别器，在似然序列尚未完全展开时，已对早期样本的结构性趋势做出了预统计判断。

这种直觉不是SPRT的替代，而是其前体信号。当直觉与SPRT冲突时，应以SPRT为准；但当直觉持续触发而SPRT尚未跨越阈值时，系统应进入高敏感度监测模式——因为人类对复杂模式的识别能力，在某些高维场景中可能暂时领先于形式化检验。

### 5.5 决策规则的认识论地位

此决策规则不是假说本身的组成部分，而是假说的**实践推论**。如果假说成立（维度混叠导致确定性失败），那么该决策规则就是在实践中应用假说的操作化工具。

此规则的效力不依赖于假说在所有条件下的普适性。只要假说在其严格界定的边界内成立，该规则就能在其适用范围内为“参数优化 vs. 策略升维”的选择提供基于证据的优先级指引。

### 5.6 递归加速：节省数千年的内在机制

如果一个人能节省十年的盲目试错，他愿意付出什么代价？

如果一种文明能节省千年的伪优化，它会把那个发明相关工具的人称作什么？

序贯检验所做的事，是把一个从未被形式化的问题——“我是否应该继续优化，还是该换一个方向”——从直觉，从经验，从走投无路，变成了一个可以被统计证据回答的问题。这不是一个工程优化工具。这是一个文明级的认知资源分配规则。

这个规则的力量，不来自它比人类直觉更聪明，而来自它能做一件人类直觉做不到的事：在旧空间的边际收益开始崩溃时——而不是在崩溃导致任务破产之后——发出可以被检验的信号。人类直觉善于优化。人类直觉不善于判断何时停止优化。

而且，这种节省不是一次性的。每一次成功破壁之后，新空间内的参数优化效率会提高，下一次碰壁的识别会更早。破壁不是线性累加，而是递归加速——每一次成功破壁，都缩短了抵达下一次破壁的时间。文明进步的模式，从“碰壁—浪费—被动切换”变为“碰壁—识别—主动破壁”。节省数千年的力量，不在于单次收益，而在于这个正反馈的递归结构。

如果您在读完引理1的证明后，认为序贯检验确实可以提前识别碰壁——那么，请您回到§1.2末尾的历史案例，重新计算一次。您会得到自己的答案。

人类文明在错误空间内过度优化的历史，是一部缺乏“何时切换”判据的历史。现在，这部历史有了被改写的可能。


## 6. 突破方法：策略升维

### 6.1 操作定义

既然维度混叠的问题在于将Q和R压缩进同一个标量函数，那么突破之道必然是**将压缩的维度重新展开**——为每一个维度建立独立的决策层，并确定它们之间的序贯关系。

> **策略升维**：将质量判断与相关性判断解耦为两个序贯独立的决策层——第一层独立执行质量维度的判断，其输出作为候选集传入第二层；第二层在候选集中独立执行相关性维度的判断。升维后的系统不再使用单一标量同时代理两个维度，而是为每个维度提供独立的决策通道，使系统能够根据环境状态动态调整各层的决策规则。

在记忆管理的具体语境中，解耦后的系统架构从：

```
所有数据 → [单标量排序] → top-K → 注入上下文
```

变为：

```
所有数据 → [质量审查] → 候选集 → [相关性排序] → top-K → 注入上下文
                ↓（未通过）
           加工器处理 → 重新审查 → 候选集 / 排除
```

- **质量审查层（资格判断）**：独立于当前目标，仅基于数据的内在属性（体积、格式、语义自足性）判断数据是否具备被注入上下文的基本条件。其决策规则可随环境状态动态调整——当窗口充裕时降低准入门槛，当窗口紧缺时收紧标准。
- **相关性管理层（激活判断）**：在通过质量审查的候选集中，独立基于数据与当前目标的关系判断数据此刻是否应被激活。注意力机制只在此层运作——它只负责颁奖，不再负责审查资格。

### 6.2 与传统方案的本质区别

传统方案的根本问题，不在于使用了多少信号或信号的权重如何，而在于**用颁奖机制同时代理了审查职能**。注意力机制的本职工作是颁奖——基于Query与Key的匹配程度，决定哪些信息应该被关注。但当系统试图用同一个注意力分数去同时代理质量判断时，它就在要求颁奖者同时判断获奖者是否具备获奖资格。策略升维的本质，就是让颁奖机制回归本位。

### 6.3 深层逻辑：范畴错误的纠正

相关性判断相对于质量判断，处于一个更高的抽象层面——它是质量判断的**元操作层**。相关性判断决定的是认知资源的分配方向：优先审查哪些数据的质量，忽略哪些数据的质量，延迟哪些数据的审查。这是**关于操作的决策**（元操作），而不是**操作本身**。

维度混叠的核心错误，是把“能不能被消费”（质量）和“该不该被考虑”（相关性）两个问题全部压平在同一个认知层面，用同一个标量函数去同时回答。这恰好是罗素在逻辑类型论（Russell, 1908）中警告的那种错误：混淆一个集合和它的元素所属的逻辑类型。将二阶操作压入一阶操作的框架内，用一阶操作的手段去代行二阶操作的职能——这不是近似，是范畴错误。

策略降维失败的深层原因，不是在操作层少了一个工具，而是没有元操作层——让颁奖机制僭越了职能，犯了范畴错误。策略升维不是“更好的工程实践”，而是纠正范畴错误的唯一操作。

### 6.4 维度盲点：脆弱性的感知根源

维度混叠系统在环境切换时必然犯错的深层原因，可以进一步追溯到比范畴错误更底层的机制：**维度盲点**。系统通过单标量函数 $h$ 感知数据，$h$ 的输出构成系统唯一的信息维度——在数学上，这是一个从高维空间（Q, R, ...）到一维空间（ℝ）的投影。在这一投影中，Q与R的独立差异可能被完全压缩（当 $h(d_A) = h(d_B)$ 时），也可能被部分保留但无法独立读取（当 $h(d_A) \neq h(d_B)$ 时，系统不知道分差来自Q还是R）。

无论哪种情况，系统对“当前需要以哪个维度为决策依据”这一元信息是完全盲的。这个盲点不是信息不足——它是一个**感知架构的维度缺失**。策略升维在操作上等价于为系统新增一个感知维度——使原先被压缩在同一个标量中的维度差异，在独立的决策层中被分别观测和处理。

### 6.5 方法的局限与诚实声明

策略升维不是万能解药。我们公开承认以下局限性：

1. **额外架构复杂性**：解耦引入了两个决策层而非一个。如果质量审查层本身设计不当，它会成为新的单一故障点。

2. **Q-R耦合场景**：在那些“格式良好”本身就等同于“有用”的任务中（如格式要求极严格的公文处理），Q和R高度耦合，维度混叠系统可能恰好够用，解耦可能不带来额外收益。

3. **实时性约束**：在毫秒级延迟要求的场景中，单标量排序的简洁性可能压倒解耦带来的鲁棒性收益。

4. **必要性而非充分性**：策略升维是消除我们所诊断的特定确定性失败模式的逻辑必要条件，但绝不是系统总体性能最优的充分条件。解耦后各层的实现质量仍是经验问题。

策略升维不承诺一劳永逸地消除所有可能的认知脆弱性。它承诺的是：就本文所严格界定的那一类由维度混叠导致的确定性失败模式而言，解耦是消除它的架构必然。若更高层级的认知空间存在新的正交维度被压缩，策略升维可递归应用（见§8.3）。

> *破壁不是更好的优化，而是让颁奖者只管颁奖，审查者只管审查。*


## 7. 理论边界与可证伪条件

一个科学假说最重要的特征，是主动划定自己的边界，并承担被证伪的风险。

### 7.1 假说的适用条件

本文假说的成立依赖于以下条件的同时满足：

1. **存在两个逻辑正交的决策维度**：质量（数据的内在可消费性）和相关性（数据与当前目标的关系属性）是独立变化的。
2. **系统处于维度混叠状态**：系统使用单一标量函数做最终排序决策，且该函数不显式包含环境状态作为独立可操作的输入通道。
3. **环境状态发生切换**：任务进行中，系统的认知负载状态在“信息饥渴”和“资源紧缺”之间切换，且这两种状态要求不同的决策优先序。
4. **参数优化已充分**：在旧架构内已经进行了充分的参数调优——不能将“调得不够好”误诊为结构性碰壁。

### 7.2 不适用场景

1. **单目标任务**：当系统有且仅有一个永久不变的目标时，相关性维度退化为常数，固定排序碰巧够用。
2. **Q和R高度耦合的任务**：在那些“格式良好”本身就是“有用”的充要条件的任务中，Q和R存在函数依赖关系，需要反转优先序的条件不成立。
3. **毫秒级实时场景**：延迟约束压倒认知完备性约束。
4. **非架构性问题**：如果失败的原因是模型能力本身不足（如推理能力不够），而非记忆管理的架构缺陷，则本文的诊断框架不适用。

### 7.3 可证伪条件

我们主动给出本假说被证伪的明确条件：

1. **核心预测的证伪**：如果有人在满足§7.1全部适用条件的任务环境中，找到一个满足定义2的维度混叠系统，其在环境状态频繁切换的长程任务中**不**表现出确定性的失败模式（即固定排序在至少一种状态下概率1犯错），且这一结果可被独立复现，那么本文的核心假说将被证伪。

2. **50%退化极限的证伪**：如果有人在严格的 $h(d_A) = h(d_B)$ 条件下，找到一个维度混叠系统能稳定超越50%准确率地区分过奖数据和屈才数据，且该系统的决策不依赖任何独立于 $h$ 的质量或相关性信息通道，那么引理2的结论将在该条件下被推翻。

3. **解耦必要性的证伪**：如果有人能证明，存在一种不依赖解耦操作的替代方法，能在§7.1的条件下稳定消除固定排序的确定性失败模式，而不引入任何独立的质量审查或环境状态感知机制，那么本文关于策略升维必要性的主张将被证伪。

4. **发现路径的可复现性**：本文声称，从工程实践到形式化理论的跃迁，是维度自由更新论在认知发现领域的一个合法实例。如果有人在以下条件被满足时，无法复现类似跃迁——(a) 先独立完成一个非平凡的工程系统；(b) 后从工程选择中归纳共同结构；(c) 最终形式化为可检验假说——则本文关于“实践优先破壁”路径的普适性主张将被削弱。此证伪条件不触及引理1-2的数学有效性，但将限制本文作为一般方法论的适用范围。

我们欢迎任何认真的证伪尝试。假说的价值不取决于它是否能永远站立，而取决于它是否能激发进一步的研究和检验。


## 8. 讨论

### 8.1 工程实践的启发

本文提供的概念工具——质量(Q)、相关性(R)、过奖、屈才、碰壁经验、次元壁——构成了一套诊断语言，帮助工程师更精确地讨论记忆管理中的架构权衡。具体而言：

1. **诊断工具**：当系统在长程任务中表现出顽固的、不可通过参数调优消除的错误模式时，可以使用Q-R框架来分析：这些错误是过奖还是屈才？是质量判断的问题还是相关性判断的问题？

2. **碰壁经验的判据**：序贯概率比检验为“是否需要考虑架构重构”提供了统计上可操作的依据，区分“真实的碰壁”和“随机的性能波动”。

3. **设计原则**：当任务要求在不同环境状态下反转决策优先序时，应考虑将质量和相关性解耦为独立的决策模块。模型切换测试可作为检验现有系统是否已隐式执行解耦的快速诊断工具。

#### 内化与扩散：从论证到效用的路径

本文提供的工具——黄色警报、红色警报、优先级倾斜规则——不是部署在服务器上的监控脚本。它们是部署在认知上的决策习惯。

黄色警报的数学定义可以在附录里找到。SPRT的阈值公式可以在参考文献里查到。但“在第三次看到准确率纹丝不动时，本能地调出似然序列”——这不是技术操作。这是内化后的本能反应。

本文的论证可以停留在纸上。本文的工具，需要被装进使用者的认知里。

一个工程师内化了这套判据，他在调试时不再盲目调参。一个团队内化了这套判据，它的季度复盘中会多一个问题：“这是参数问题还是空间问题？”一个领域内化了这套判据，碰壁的识别就不再依赖于个别天才的偶然洞察，而成为领域内部的制度化反应。

内化的规模，决定了效用的规模。

效用不是从论文发表开始的。它是从第一个使用者开始用黄色警报来诊断自己的停滞，而不是用“再试一组参数”来回避它时，才开始的。

> *本小节不构成论证的必要部分。它只是提醒：读完这篇论文之后，这套工具的去向，取决于您。*
>
> 关于碰壁-破壁循环在更普遍认知运动中的递归加速效应，以及AI内化的可能性，见《维度自由更新论》。

### 8.2 与信息论的结构对应（推测性延伸）

一个值得探索的方向是本文框架与香农信息论的结构性对应关系。

在最极端的情况下，当单标量函数将Q和R的差异完全抹平时（$h(d_A) = h(d_B)$），系统从该标量输出中获得的关于决策维度的互信息为零。此时准确率退化至50%——这正是零互信息信道在二元假设检验中准确率上限的信息论等价表述。

更一般地，维度混叠操作可以被建模为一个认知信道：输入为高维的(Q, R)空间，信道操作为投影至一维标量 $h$，输出为该标量的噪声观测。在此模型中，系统感知环境状态的能力对应信道的边信息。当边信息为零时，信道容量退化至零互信息极限，准确率上限退化为50%。当边信息完整时，准确率趋近100%。

这一对应关系提示我们，本文的核心论证可能与信息论中信道容量的退化是同一个数学结构在认知领域的投影。如果这一连接能被严格建立，香农信息论的工具箱（信道容量公式、率失真函数、最优编码定理）可能为认知架构的脆弱性分析提供系统的数学基础。

**但我们必须坦率声明**：此连接目前在本文中是推测性的。认知信道是否能被严格证明为物理信道的子类，互信息的计算在认知域中如何被操作化，这些问题需要后续独立的形式化工作来回答。

### 8.3 递归开放性：下一次破壁在哪里？

策略升维不是一次性操作。当认知系统面临新的正交维度被压缩时，升维操作本身可以被再次应用。

**递归升维的终止判据**：递归在第 $k$ 层终止，当且仅当在第 $k$ 层破壁后产生的 $H_k$ 内，参数优化的后验预测分布在任务要求的容忍范围内稳定收敛。终止判据绑定于一个可观测的经验事实——新空间是否能够稳定地解释证据——而非由人任意设定的外部阈值。它防止了“为破壁而破壁”。

**升维深度的决定原理**：某次具体破壁的深度，不由理论的“彻底性”决定，而由满足终止判据所需的被压缩正交维度消解程度决定。同一个认知系统在不同任务中会自然展现出不同的破壁深度——任务要求越高，所需消解的压缩维度越多，破壁越深。

**终止的内生化**：每一次策略升维都支付两种成本——新决策层的维护成本和层间通信成本。同时产生一种收益——新展开的维度所释放的决策信息增量。当边际收益低于边际成本时，递归在原理上终止。递归的终止不是被宣布的，而是被升维自身的成本结构所决定的。

**一个尚未被识别的被压缩维度**：信任管理可能是一个值得关注的候选。信任不是安全（权限模型），不是可解释（状态透明），不是可靠（错误率低），而是一个独立于这三者的、系统与用户之间的一维连续量，随交互历史动态更新。当前隐式代理信任判断的混合信号——显式确认的频率、配置修改的审批严格度、卡死检测的敏感阈值——是否构成了新一轮的维度混叠，是留给后续工作的开放问题。

### 8.4 理论体系展望：从领域假说到普遍原理

本文的主体定位为一个可证伪的领域假说。然而，如果本假说的核心预测在后续独立检验中得以幸存——即维度混叠导致的确定性失败被反复验证——那么它将与解耦引理（附录C）及维度自由更新论构成一个三层嵌套的理论结构：

- **元操作层：解耦引理**。该引理已被证明，确立了策略可指涉性的最低架构要求：任何处于混叠状态的认知系统，若欲优化其策略，必须先执行数据逻辑解耦。它解释了维度混叠为何在主流框架中如此普遍——系统从未将“策略”本身作为可审视的独立实体。

- **领域原理层：破壁假说**。即本文的核心主张，尚待独立检验。它断言：在解耦后的系统中，若决策层再度压缩正交维度，则系统在环境状态切换时必然表现出确定性的决策僵化。突破该脆弱性的方法是策略升维。

- **普遍理论层：维度自由更新论**。如果破壁假说被反复验证，那么它与策略升维的递归开放性将构成维度自由更新论在认知架构层的一个领域实例——后者主张认知更新不应受固定维度空间的限制，“碰壁—破壁—新空间内参数优化恢复收敛”是认知运动的普遍形式。本文的黄色警报、红色警报与优先级倾斜规则，正是该普遍运动中“碰壁识别”与“破壁决策”两个环节在认知架构领域的操作化实例。

我们在此记录这个三层结构，不是作为本文已经确立的结论，而是作为未来研究的一个结构化路线图。它邀请社区共同检验：从一个领域假说到普遍原理的跃迁，是否可能沿着这条路径逐层推进。

### 8.5 跨领域启发性延伸

本节内容为推测性延伸，不构成前文论证的必要部分。以下延伸旨在展示本文概念框架的潜在解释力半径，其有效性需要通过各自领域的独立研究来检验。

#### 8.5.1 教育评价

传统教育评价体系可以被类比为一个维度混叠系统——标准化考试分数同时代理“天赋识别”和“资源分配”。在长程（终身学习）和多目标（多元智能）条件下，该系统在需要反转评价优先序的特定场景中，可能表现出类似“过奖”（高分低能者获得更多高阶资源）和“屈才”（低分高能者被系统性闲置）的系统性错误。这是一个结构上的类比，其有效性依赖于教育评价系统与我们形式化模型之间的结构同构性。

#### 8.5.2 文字系统

拼音文字将书写形式绑定为发音的函数，这可以类比为将语义记录的稳定性需求和语音记录的流变性事实压缩进同一符号系统。在长程时间尺度和多目标需求的条件下，拼音文字系统在“稳定记录”和“精准记音”之间的权衡承受着结构性代价——拼写不规则性与历史文献解读困难是其可观测的症状。汉字的形声分离——形旁提供语义锚点，声旁提供非绑定的发音提示——则是一次史前的“解耦”操作。本节的分析不构成对任何语言或文明的优劣评判，而是展示破壁原理在文化工具演化领域的启发性解释力。如果这一类比在未来经过语言学领域的独立严格验证，它将为“语言学上升为工程学”——即从描述语言现象到诊断文字系统的架构缺陷——提供一个数学上严格的起点。

## 附录A：错误穷举定理的形式化证明

**定理1（错误穷举定理）** 若一个系统的上下文窗口 $C_t$ 在时刻 $t$ 未能包含最优信息组合 $C^*$，则该失败必然可以归因于质量函数的至少一个错误或相关性函数的至少一个错误。

*证明*：设存在数据 $d \in C^*$ 但 $d \notin C_t$（应纳入而未纳入）。若质量函数判定 $Q(d)=0$，则该错误由质量函数导致——系统错误地将一条本应通过质量门槛的数据排除在外。此为**假阴性能力错配**（屈才）。若质量函数判定 $Q(d)=1$ 但相关性函数判定 $R_t(d)=0$，则该错误由相关性函数导致——系统在候选集中错误地未触发这条数据。此为**假阴性意图失焦**。

设存在数据 $d \in C_t$ 但 $d \notin C^*$（不应纳入而纳入）。若质量函数判定 $Q(d)=0$，则该错误由质量函数导致——系统错误地将一条不应通过质量门槛的数据纳入上下文。此为**假阳性能力错配**（过奖）。若质量函数判定 $Q(d)=1$ 且相关性函数判定 $R_t(d)=1$，但 $d \notin C^*$，则该错误由相关性函数导致——系统在候选集中错误地触发了这条数据。此为**假阳性意图失焦**。

四种情况覆盖 $C_t \neq C^*$ 的所有可能，且每种情况都可以被唯一地归因于质量函数或相关性函数的错误。□


## 附录B：实验代码

（完整实验代码可在任意 Python 3.9+ 环境中运行，依赖 `numpy` 和 `statsmodels`。）

```python
import numpy as np
from statsmodels.stats.contingency_tables import mcnemar

np.random.seed(42)

def generate_tasks(n_tasks=30):
    tasks = []
    for _ in range(n_tasks):
        lowQ_highR = np.array([0., 1.])  # 质量0，相关性1
        highQ_lowR = np.array([1., 0.])  # 质量1，相关性0
        tasks.append((lowQ_highR, highQ_lowR))
    return tasks

def run_experiment(tasks, n_episodes=30, switch_after=15):
    states = ['hungry'] * switch_after + ['scarce'] * (n_episodes - switch_after)
    correct_docs = [tasks[i][0] if s == 'hungry' else tasks[i][1]
                    for i, s in enumerate(states)]

    def fixed_linear(w_q, w_r, docs, state):
        scores = w_q * docs[:, 0] + w_r * docs[:, 1]
        return docs[np.argmax(scores)]

    def state_aware_strategy(docs, state):
        if state == 'hungry':
            scores = docs[:, 1]
        else:
            scores = docs[:, 0]
        return docs[np.argmax(scores)]

    # 网格搜索最佳固定线性
    best_w = None
    best_err = float('inf')
    for w_q in np.arange(0., 1.01, 0.1):
        w_r = 1 - w_q
        err = 0
        for i, (docs_pair, state) in enumerate(zip(tasks[:n_episodes], states)):
            docs = np.array(docs_pair)
            chosen = fixed_linear(w_q, w_r, docs, state)
            if not np.array_equal(chosen, correct_docs[i]):
                err += 1
        if err < best_err:
            best_err = err
            best_w = (w_q, w_r)

    strategies = {
        "随机基线": lambda docs, s: docs[np.random.randint(0, 2)],
        "固定线性 (0.0Q,1.0R)": lambda docs, s: fixed_linear(0.0, 1.0, docs, s),
        "固定线性 (1.0Q,0.0R)": lambda docs, s: fixed_linear(1.0, 0.0, docs, s),
        "固定线性 (0.5Q,0.5R)": lambda docs, s: fixed_linear(0.5, 0.5, docs, s),
        f"最佳固定线性 (Q={best_w[0]:.1f},R={best_w[1]:.1f})":
            lambda docs, s, wq=best_w[0], wr=best_w[1]: fixed_linear(wq, wr, docs, s),
        "策略升维": lambda docs, s: state_aware_strategy(docs, s),
    }

    results = {}
    scarce_correct = {}
    for name, strat in strategies.items():
        err_hungry = err_scarce = 0
        total_hungry = total_scarce = 0
        scarce_correct_list = []
        for i, (docs_pair, state) in enumerate(zip(tasks[:n_episodes], states)):
            docs = np.array(docs_pair)
            chosen = strat(docs, state)
            correct = correct_docs[i]
            if state == 'hungry':
                total_hungry += 1
                if not np.array_equal(chosen, correct):
                    err_hungry += 1
            else:
                total_scarce += 1
                is_correct = np.array_equal(chosen, correct)
                scarce_correct_list.append(is_correct)
                if not is_correct:
                    err_scarce += 1
        results[name] = (err_hungry, total_hungry, err_scarce, total_scarce)
        scarce_correct[name] = scarce_correct_list
    return results, scarce_correct, best_w, best_err

tasks = generate_tasks(30)
results, scarce_correct, best_w, best_err = run_experiment(tasks)

print("—" * 20)
print(f"最佳固定线性权重: Q={best_w[0]:.1f} R={best_w[1]:.1f}  总错误 = {best_err}")
print("—" * 20)
print("破壁原理仿真实验（基于标签得分，正确选择互斥）")
print("Hungry 状态正确文档: (Q=0,R=1)")
print("Scarce 状态正确文档: (Q=1,R=0)")
print("容量=1，正确选择互斥")
print("—" * 20)

for name, (eh, th, es, ts) in results.items():
    print(f"{name:35s}  Hungry: {eh}/{th} ({eh/th:.0%})   Scarce: {es}/{ts} ({es/ts:.0%})")

linear_key = f"最佳固定线性 (Q={best_w[0]:.1f},R={best_w[1]:.1f})"
optimal_key = "策略升维"
linear_correct = scarce_correct[linear_key]
optimal_correct = scarce_correct[optimal_key]

both = sum(1 for l, o in zip(linear_correct, optimal_correct) if l and o)
linear_only = sum(1 for l, o in zip(linear_correct, optimal_correct) if l and not o)
optimal_only = sum(1 for l, o in zip(linear_correct, optimal_correct) if not l and o)
neither = sum(1 for l, o in zip(linear_correct, optimal_correct) if not l and not o)

table = [[both, linear_only], [optimal_only, neither]]
p_val = mcnemar(table, exact=True).pvalue if (linear_only + optimal_only) > 0 else 1.0

print("\n" + "—" * 20)
print("McNemar检验 (Scarce状态下)")
print(f"            升维正确  升维错误")
print(f"线性正确      {both}        {linear_only}")
print(f"线性错误      {optimal_only}        {neither}")
print(f"p-value = {p_val:.6f}")
if p_val < 0.05:
    print("结论: 策略升维显著优于最佳固定线性 (p<0.05)")
else:
    print("结论: 差异不显著")
print("—" * 20)
```


## 附录C：解耦引理——维度混叠普遍性的深层根源

注：本附录保留“策略降维”这一历史术语，其含义与正文中的“维度混叠”一致。此处使用旧称是因为解耦引理在理论发展史上先于维度混叠概念被提出。

维度混叠为何在主流框架中如此普遍？答案在于一个更基础的架构缺陷——数据与逻辑的混叠。

**定义C.1（混叠状态与解耦状态）** 设认知系统 $\Sigma$ 的状态由三元组 $(X, F, Y)$ 描述，其中 $X$ 为输入空间，$Y$ 为输出空间，$F: X \to Y$ 为策略函数。系统处于**混叠状态**当且仅当在其可观测表示中，$F$ 不作为独立实体存在，仅存储数据对 $\{(x_i, y_i) \mid y_i = F(x_i)\}$ 的集合。系统处于**解耦状态**当且仅当其架构显式包含策略库 $\mathcal{F} = \{F_1, \ldots, F_k\}$（数据层）与选择变量 $c$（逻辑层）作为独立组件。

**解耦引理**：对于任何处于混叠状态的认知系统，若其目标是优化策略 $F$，则其必须先执行解耦操作，进入解耦状态。在混叠状态中，不存在可被优化的“策略”实体。

*直观论证*：在混叠状态中，策略 $F$ 仅通过其输入-输出对隐式呈现。存在无限多个函数与已有数据一致，系统无法唯一确定策略本身，因此无法评估、比较或搜索更好的策略。优化在概念上不可定义。

传统记忆管理方案从未执行此解耦，故维度混叠在其架构中不可见、不可避。这正是正文§2.4所观察到的——所有主流框架都在事实上执行了维度混叠，却从未意识到自己做了这个选择。解耦引理解释了这种集体盲区的深层根源。
