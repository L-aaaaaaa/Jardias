# 运行和开发提示

## 环境要求

- Python 3.10+
- 至少一个 LLM （本项目称IPU）供应商的 API Key

## 依赖

```bash
pip install -r requirements.txt
```

依赖清单：

| 包 | 用途 |
|---|---|
| `openai` | 多供应商统一 API 客户端（DeepSeek / MiniMax / Qwen 均兼容 OpenAI 协议）|
| `loguru` | 终端彩色日志 + 文件轮转日志 |
| `pydantic` | 数据模型校验（ActorConfig 等） |

## 快速开始

### 1. 设置 API Key（至少一个）

```powershell
# Windows PowerShell
$env:MINIMAX_API_KEY = "your-key"
$env:DASHSCOPE_PLAN_API_KEY = "your-key"
$env:DEEPSEEK_API_KEY = "your-key"
```

### 2. 启动

```bash
# 进入交互菜单（选择/创建角色）
python app.py

# 直接启动指定角色
python app.py --character default --provider minimax --ipu 2.7

# 列出所有可用角色
python app.py --list
```

### 3. 基础对话

```
当前角色: default | 引擎: minimax/2.7
输入 'quit' 退出，输入 'switch' 切换角色

> 你好，请介绍一下自己
（AI 回复…）

> ··其他角色名 你好
（切换到其他角色并发送消息）

> quit
```

## 运行方式详解

### CLI 参数

```
python app.py [--provider <名称>] [--ipu <简称>] [--character <角色名>] [--list]
```

| 参数 | 说明 | 默认值 |
|---|---|---|
| `--provider` | 供应商名称 | `minimax` |
| `--ipu` | 智能基元简称（短名） | `2.7` |
| `--character` | 启动角色名，不指定则进入交互菜单 | 无（进入菜单） |
| `--list` | 列出所有角色并退出 | — |

### 可用引擎

| 供应商 | 简称 | 模型 ID | 能力 |
|---|---|---|---|
| minimax | `2.7快` | MiniMax-M2.7-highspeed | text, tool, thinking, fast |
| minimax | `2.7` | MiniMax-M2.7 | text, tool, thinking |
| minimax | `M3` | MiniMax-M3 | text, tool, thinking, vision, long_ctx, reasoning |
| dashscope | `千问3.6+` | qwen3.6-plus | text, tool, vision, thinking |
| dashscope | `kimi 2.5` | kimi-k2.5 | text, tool, vision, long_ctx |
| dashscope | `glm-5` | glm-5 | text, tool, thinking |
| dashscope | `M2.5` | MiniMax-M2.5 | text, tool |
| deepseek | `v4-flash` | deepseek-v4-flash | text, tool, thinking, reasoning, fast |

完整配置见 `yinao/providers.json5`。

### ·· 快速路由

直接在输入框使用 `··` 前缀，绕过 LLM 决策直接执行：

```
··角色名 消息内容        → 切换到指定角色并发送消息，注意名字后的空格是生效条件
··工具名                 → 直接调用工具
```

示例：

```
> ··小明 帮我分析一下这个数据
（切换到角色"小明"，发送"帮我分析一下这个数据"）

> ··shice_schedule_list
（直接列出所有时策任务）
```

## 开发指南

### 添加新的 LLM 供应商

1. 在 `yinao/ipu_client/` 创建 `<provider>.py`，实现 `chat_fn(messages, config, character_name, on_history_save) -> ChatResult`
2. 在 `yinao/ipu_client/__init__.py` 的 `resolve_chat()` 中注册分支
3. 在 `yinao/providers.json5` 添加供应商 + 模型条目
4. 在 `yinao/ipu_resolver.py` 的 `IPU_REGISTRY` 中注册模型简称

### 添加新工具

**普通工具**（确定性逻辑）：

```python
# 在 tool/builtin.py 中
def my_tool_handler(args: dict) -> str:
    # 纯 Python 逻辑
    return "result"

_BUILTIN_HANDLERS["my_tool"] = my_tool_handler

tools.register(ToolDef(
    name="my_tool",
    description="工具描述，LLM 据此判断何时调用",
    parameters={
        "type": "object",
        "properties": {
            "param1": {"type": "string", "description": "参数说明"}
        },
        "required": ["param1"]
    },
    fn=my_tool_handler,
))
```

**LLM 工具**（需要推理能力的工具）：

```python
# 在任何模块中
from tool.llm_tool import llm_tool

@llm_tool(
    ipu="v4-flash",           # 使用哪个 IPU 执行
    output_schema={"key": "string", "items": "array of string"},
    system="你是 XXX 分析器，…"
)
async def my_llm_tool(input_text: str) -> dict:
    pass  # 由装饰器自动生成实现
```

### 创建新角色

启动时选择交互菜单的"创建新角色"，填写：

- 角色名（文件目录名，仅英文/数字/下划线）
- 角色标题（中文显示名）
- 角色描述（traits，注入 identity）
- 供应商 + 引擎选择

或者直接手动在 `character/<角色名>/` 下放置配置文件。

## 常见问题

**Q: 报错 "API key not found"？**
A: 设置对应供应商的环境变量。至少需要一个：`MINIMAX_API_KEY`、`DASHSCOPE_PLAN_API_KEY` 或 `DEEPSEEK_API_KEY`。

**Q: 模型调用失败怎么处理？**
A: 框架有自动 fallback 机制——当前供应商不可用时，自动尝试其他可用供应商。故障判定走熔断器（circuit_breaker），连续失败会暂时避开该供应商。

**Q: 如何切换模型？**
A: 对话中直接说"切换到 deepseek v4-flash"，LLM 会调用 `update_runtime` 工具完成热切换，无需重启。

**Q: history.json 会不会无限增长？**
A: 原始 history 会，但发给 LLM 的上下文始终是 O(1)（4 段固定消息结构，通过金字塔压缩控制）。
